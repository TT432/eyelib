## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Command source → MC command adapter | User-supplied command text crosses into Brigadier parsing and suggestion handling. |
| MC command adapter → particle runtime/request seam | `ResourceLocation` and command source state must be flattened to strings/floats before platform-free helpers. |
| Command adapter → network packet | Spawn id, particle id, and position become transport payload state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | `EyelibParticleCommand.suggestEffects` | mitigate | Filter suggestions through `isValidResourceLocation` and source active ids from `ParticleLookup.names()` only. |
| T-13-02 | Denial of Service | `EyelibParticleCommand.execute` | accept | Existing command remains server-player-only and returns `0` for non-player sources; Phase 13 does not add rate limiting or permissions per D-04. |
| T-13-03 | Information Disclosure | success message | accept | Message contains only user-invoked particle id and coordinates already supplied/derived by command source; no additional state is exposed. |
</threat_model>
