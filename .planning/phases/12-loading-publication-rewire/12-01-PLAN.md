<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resource-pack JSON → particle module parser | Untrusted resource JSON enters importer codec and runtime conversion. |
| source metadata → active registry | Resource/file ids are untrusted metadata and must not become active lookup keys. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering | `ParticleResourcePublication` | mitigate | Parse through importer `BrParticle.CODEC` and `ParticleDefinitionAdapter`; publish only successful `ParticleDefinition` values. |
| T-12-02 | Spoofing | active particle keys | mitigate | Key active store only by `ParticleDefinition.identifier()`; tests assert source keys are absent. |
| T-12-03 | Repudiation | conversion failures | mitigate | Record failed source ids/messages in `ParticleLoadReport` and log source id + error. |
| T-12-04 | Denial of Service | one invalid resource in reload set | mitigate | Skip invalid resources while still replacing with valid entries, matching existing reload resilience. |
</threat_model>
