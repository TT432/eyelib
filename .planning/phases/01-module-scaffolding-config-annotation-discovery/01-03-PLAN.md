```toml
A standalone client-side smoke testing framework for Minecraft Forge mods.
Tests are discovered via @ClientSmoke annotation using bytecode-level ASM scanning
(zero class loading), executed inside an auto-created creative flat world, captured
as clean screenshots, and reported as JSON — all config-driven.

This mod is client-only and will not load on dedicated servers.
'''

[[dependencies.clientsmoke]]
modId="forge"
mandatory=true
versionRange="${forge_version_range}"
ordering="NONE"
side="CLIENT"

[[dependencies.clientsmoke]]
modId="minecraft"
mandatory=true
versionRange="${minecraft_version_range}"
ordering="NONE"
side="CLIENT"
```

**Critical design decisions:**
- `modId="clientsmoke"` — must match `ClientSmokeMod.MOD_ID` exactly (Pitfall 6)
- `side="CLIENT"` on ALL dependency blocks — per Pitfall 13, this mod is client-only; declaring `side="BOTH"` would cause Forge to attempt loading on dedicated servers, crashing with `NoClassDefFoundError` for Minecraft client classes
- `${loader_version_range}`, `${forge_version_range}`, `${minecraft_version_range}` — these are expanded by the `processResources` task in `eyelib-clientsmoke/build.gradle` (created in Plan 01 Task 2), matching the values in `gradle.properties` (`forge_version_range=[47.1.3,)`, `minecraft_version_range=[1.20.1,1.21)`)
- `${mod_version}`, `${mod_license}`, `${mod_authors}` — also expanded by processResources, inherited from root `gradle.properties`
- No `[[dependencies.clientsmoke]]` block for the annotation module — the annotation is `compileOnly` (not a mod dependency). It's consumed as a library, not as a Forge mod
- `description` uses triple-quoted TOML string for multi-line — matches existing eyelib mods.toml patterns
