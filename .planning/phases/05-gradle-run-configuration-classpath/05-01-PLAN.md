Two changes in the root `build.gradle` file, both implementing GRAD-01, GRAD-02, and GRAD-03:

**Change A — Add `clientSmoke` run config (GRAD-01, GRAD-02):**

Insert a `clientSmoke` block inside `legacyForge.runs`, immediately after the `client { ... }` block and before `server { ... }`. The block content:

```groovy
        clientSmoke {
            client()
            gameDirectory = project.file('run/clientsmoke')
            systemProperty 'forge.enabledGameTestNamespaces', 'clientsmoke'
        }
```

- `client()` — copies the default client run behavior (main class, JVM args, working directory from `configureEach`)
- `gameDirectory = project.file('run/clientsmoke')` — isolates smoke test game state from normal `run/` (GRAD-02)
- `systemProperty 'forge.enabledGameTestNamespaces', 'clientsmoke'` — enables Forge game test scanning for the `clientsmoke` mod namespace

Do NOT remove or modify the existing `client` block or any other run config blocks. The `configureEach` block already applies to all runs (including `clientSmoke`) and provides logging config — no changes needed there.

**Change B — Remove conditional gate on `eyelib-clientsmoke` localRuntime (GRAD-03):**

In the `dependencies` block, locate the following lines:

```groovy
    if (project.findProperty('enableSmokeTest') == 'true') {
        localRuntime project(':eyelib-clientsmoke')
    }
```

Replace with the unconditional form:

```groovy
    localRuntime project(':eyelib-clientsmoke')
```

This makes `eyelib-clientsmoke` always present on every `localRuntime` classpath (runClient, runClientSmoke, runServer, etc.). The mod's actual behavior is controlled at runtime by `ClientSmokeConfig.ENABLED` (default: `false`), so it remains idle by default on non-smoke run configs. Phase 6 will add system property injection to enable it specifically for `runClientSmoke`.

**Important:** Do NOT move this line outside the `dependencies` block. It stays exactly where the `if` block was, preserving its position relative to other dependency declarations.
