**Method 1: `isEnabled()` (OVRD-01):**

```java
/**
 * Returns whether the smoke test framework should be active.
 *
 * <p>Priority order (per OVRD-01):
 * <ol>
 *   <li>System property {@code clientsmoke.enabled} — set by Gradle run config</li>
 *   <li>Fallback to {@link #ENABLED} ForgeConfigSpec value (default: {@code false})</li>
 * </ol></p>
 *
 * <p>When the system property is set to any value, {@link Boolean#parseBoolean(String)}
 * determines the result ({@code "true"} → true, anything else → false).
 * When the system property is absent ({@code null}), falls back to TOML config.</p>
 *
 * @return true if smoke testing should be enabled
 */
public static boolean isEnabled() {
    String prop = System.getProperty("clientsmoke.enabled");
    if (prop != null) {
        return Boolean.parseBoolean(prop);
    }
    return ENABLED.get();
}
```

**Method 2: `shouldExitAfterSmoke()` (OVRD-02):**

```java
/**
 * Returns whether the client should automatically exit after smoke tests complete.
 *
 * <p>Priority order (per OVRD-02):
 * <ol>
 *   <li>System property {@code clientsmoke.autoExit} — set by Gradle run config</li>
 *   <li>Fallback to {@link #EXIT_AFTER_SMOKE} ForgeConfigSpec value (default: {@code true})</li>
 * </ol></p>
 *
 * <p>Parse semantics identical to {@link #isEnabled()}.</p>
 *
 * @return true if the client should exit automatically after tests
 */
public static boolean shouldExitAfterSmoke() {
    String prop = System.getProperty("clientsmoke.autoExit");
    if (prop != null) {
        return Boolean.parseBoolean(prop);
    }
    return EXIT_AFTER_SMOKE.get();
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gradle `systemProperty()` → JVM `System.getProperty()` | Build-time property injection crosses into runtime config. Trusted boundary — the build script is version-controlled. |
| `System.getProperty()` → `ClientSmokeConfig.isEnabled()` | JVM property read within the mod. No parsing risk beyond `Boolean.parseBoolean()`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | `build.gradle` `systemProperty` values | accept | Build scripts are version-controlled; tampering is standard supply-chain concern. No runtime exposure to untrusted input. |
| T-06-02 | Information Disclosure | System property values | accept | Property values (`true`/`false`) carry no secrets. Properties are visible in process listing but contain no sensitive data. |
| T-06-03 | Elevation of Privilege | `Boolean.parseBoolean()` on untrusted property | accept | The system property is set by Gradle (trusted). If a malicious actor can set JVM system properties, they already own the process. No privilege boundary crossed. |
</threat_model>
