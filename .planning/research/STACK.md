# Technology Stack — ClientSmoke v1.1 Gradle Automation

**Project:** Client Smoke Test Gradle Task Automation  
**Researched:** 2026-05-08  
**Confidence:** HIGH

## Recommended Stack

### Core Framework

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| NeoForge ModDevGradle (LegacyForge) | 2.0.91 | Run configuration definition, JVM argument injection, IDE integration | Already the project's build plugin. Provides `legacyForge.runs { }` DSL with `systemProperty()`, `loadedMods`, `ideName` — exactly the primitives needed for a one-click smoke test task. No alternative needed. |
| Gradle | 8.x (wrapper-managed) | Build orchestration, task dependency management | Already the project's build system. `runClientSmoke` task is auto-generated by MDGL from run config declaration. |

### Runtime (Inside Minecraft JVM)

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Java System Properties | JDK 17 built-in | Config override bridge between Gradle and Minecraft process | `System.getProperty("clientsmoke.enabled")` is the simplest possible cross-process communication mechanism. No serialization, no network, no file I/O. The Gradle task sets `-D` flags; the mod reads them. |
| ForgeConfigSpec (Forge 1.20.1) | 47.1.3 | Persistent TOML-based config (fallback path) | Already used by v1.0 `ClientSmokeConfig`. Retained as fallback for manual invocation. System property takes priority when present. |
| SLF4J / Logback | 1.20.1 Forge bundled | Console logging for test progress and results | Already the project's logging facade. Enhanced formatting for CLI-friendly output requires no new dependency. |

### Test Artifact Generation

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Gson | Forge 1.20.1 bundled | JSON report serialization | Already used by v1.0 `handleReport()`. No change needed. |
| `javax.xml.stream` (JDK) | JDK 17 built-in | JUnit XML report generation | JDK built-in StAX parser/writer — no external dependency. Produces standard JUnit schema XML consumable by Jenkins, GitHub Actions, TeamCity. |

### No New Dependencies

The v1.1 automation layer introduces **zero new external dependencies**. Every technology used is either:
- Already present in the project (MDGL, Forge, Gson, SLF4J)
- JDK built-in (`System.getProperty()`, `javax.xml.stream`)

This keeps the dependency graph unchanged and avoids version conflict risks.

## Alternatives Considered

| Category | Recommended | Alternative | Why Not |
|----------|-------------|-------------|---------|
| Config override mechanism | `System.getProperty()` in `ClientSmokeConfig` | Forge `ModConfig` programmatic write | Writing TOML files from Gradle is fragile: race conditions, git pollution, persistent state leak. System properties are process-scoped and temporary. |
| Gradle task type | MDGL auto-generated `runClientSmoke` from `legacyForge.runs` | Custom `JavaExec` task | MDGL handles Minecraft classpath assembly, AT processing, mapping resolution, and IDE integration — reinventing this in a custom task would be massive scope creep and fragile. |
| Cross-process comm | JVM `-D` system properties | Environment variables | Both work, but `systemProperty()` is the MDGL-native API. Environment variables require `environment()` which is less idiomatic for Gradle→Minecraft. |
| Classpath inclusion | Always include `eyelib-clientsmoke` via `localRuntime` | Keep gated behind `enableSmokeTest` | If gated, the `runClientSmoke` task would need to dynamically modify the classpath, which MDGL doesn't support on a per-run basis. Always-on inclusion with `ClientSmokeConfig.ENABLED=false` as the safety net is cleaner. |
| Exit code for failures | `Runtime.getRuntime().halt(1)` on test failure | `System.exit(1)` | `halt()` is already the v1.0 design (bypasses shutdown hooks that can hang modded Minecraft). Consistent with existing architecture. |
| Test filtering transport | JVM `-D` system properties | Gradle `-P` project properties + task configuration | JVM properties reach the mod process; Gradle `-P` properties stay in the Gradle process. Both can be set on command line (`-D` vs `-P`), but `-D` is the correct transport into the forked JVM. |

## Installation / Build Script Changes

### Root `build.gradle` — new run config

```groovy
legacyForge {
    runs {
        // Existing: normal client run (unchanged)
        client {
            client()
            systemProperty 'forge.enabledGameTestNamespaces', project.mod_id
        }

        // NEW: smoke test run
        clientSmoke {
            client()
            ideName = "Run Client Smoke Tests"
            
            // Force-enable smoke testing regardless of TOML config
            systemProperty 'clientsmoke.enabled', 'true'
            systemProperty 'clientsmoke.exitAfterSmoke', 'true'
            
            // Optional: override output directory for CI
            // systemProperty 'clientsmoke.reportDir', 'build/smoke-reports'
        }
        
        // ... existing server, gameTestServer, data runs unchanged
    }
}
```

### Root `build.gradle` — dependency adjustment

```groovy
// BEFORE (v1.0):
compileOnly project(':eyelib-clientsmoke-annotation')
if (project.findProperty('enableSmokeTest') == 'true') {
    localRuntime project(':eyelib-clientsmoke')
}

// AFTER (v1.1):
compileOnly project(':eyelib-clientsmoke-annotation')
localRuntime project(':eyelib-clientsmoke')  // Always on classpath; gated by config
```

### `ClientSmokeConfig.java` — system property overrides

```java
// New static helpers (added to existing class)
private static final String PROP_ENABLED = "clientsmoke.enabled";
private static final String PROP_EXIT = "clientsmoke.exitAfterSmoke";

public static boolean isEnabled() {
    String prop = System.getProperty(PROP_ENABLED);
    if (prop != null) return Boolean.parseBoolean(prop);
    return ENABLED.get();  // fallback to TOML
}

public static boolean shouldExitAfterSmoke() {
    String prop = System.getProperty(PROP_EXIT);
    if (prop != null) return Boolean.parseBoolean(prop);
    return EXIT_AFTER_SMOKE.get();  // fallback to TOML
}
```
