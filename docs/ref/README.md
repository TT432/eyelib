# 参考资料索引

## 目的

`docs/ref/` 用来沉淀“外部事实标准”，重点回答两类问题：

1. Minecraft Bedrock Add-On 官方与社区普遍承认的结构是什么。
2. 这些结构里到底有哪些字段、目录、文件族与容器形态。

这里的文档不是项目实现说明，也不是功能设计文档，而是“参考标准整理”。

## 当前主题

- `bedrock-addon/`：Bedrock Add-On 的容器、manifest、包结构、资源族与真实样例参考。

## 编写规则

1. 优先记录“事实”和“来源”，不要先写项目结论。
2. 官方文档与社区文档分开标注，不混写成同等权威。
3. 真实 addon 样例只作为“事实调研样本”，不直接等同于规范。
4. 每个结论尽量能回溯到 `来源清单.md` 中的 URL。
