# Bedrock Add-On 行为文件层：表类资源

## 1. 适用范围

本页把 Behavior Pack 中若干“表类/规则类”资源先集中成一页：

- `blocks`
- `items`
- `recipes`
- `loot_tables`
- `trading`
- `spawn_rules`
- `structures`

后续如果某一族证据足够多，再继续拆成独立页。

> 现在递归证据已经足够多。字段/子字段级索引见：`递归字段索引-bp-entity-table.md`。

## 2. 位置矩阵

| 资源族 | 路径 | 类型 | 规范性 | 主要参考 | 备注 |
|---|---|---|---|---|---|
| blocks | `blocks/*.json` | 目录族+文件 | stable | O8-BLOCKS, O8-COMPPACK | block 行为定义 |
| items | `items/*.json` | 目录族+文件 | stable | O8-ITEMS, O8-COMPPACK | item 行为定义 |
| recipes | `recipes/*.json` | 目录族+文件 | stable sample | O8-COMPPACK | 配方定义 |
| loot_tables | `loot_tables/**/*.json` | 目录族+文件树 | stable | O8-LOOTTRADE, O8-COMPPACK | 允许子目录组织 |
| trading | `trading/**/*.json` | 目录族+文件树 | stable sample | O8-LOOTTRADE, O8-COMPPACK | 允许子目录组织 |
| spawn_rules | `spawn_rules/*.json` | 目录族+文件 | stable | O8-SPAWN, O8-COMPPACK | 生成规则定义 |
| structures | `structures/*.mcstructure` | 目录族+文件 | stable sample | O8-COMPPACK | 结构文件不是 JSON |

## 3. 顶层结构矩阵

| 资源族 | 顶层对象/关键结构 | 作用 | 规范性 | 主要参考 | 备注 |
|---|---|---|---|---|---|
| blocks | `minecraft:block` | block 行为定义 | tutorial/reference hybrid | O8-BLOCKS | 结构较清晰 |
| items | `minecraft:item` | item 行为定义 | tutorial/reference hybrid | O8-ITEMS | 结构较清晰 |
| recipes | 配方类型对象（随 recipe 类型变化） | recipe 定义 | stable sample | O8-COMPPACK | 建议后续独立拆分时再细分类型 |
| loot_tables | `pools` | loot table 根结构 | tutorial/reference hybrid | O8-LOOTTRADE | 最核心顶层字段 |
| trading | `tiers` / `groups` / `trades` 等结构 | trade table 根结构 | tutorial/reference hybrid | O8-LOOTTRADE | 当前先写成概念级顶层结构 |
| spawn_rules | `minecraft:spawn_rules` | 生成规则定义 | reference/example hybrid | O8-SPAWN | 正式顶层对象 |
| structures | `.mcstructure` 二进制结构文件 | 结构资源 | stable sample | O8-COMPPACK | 不属于 JSON 资源族 |

## 4. 主要区段矩阵

| 资源族 | 主要区段/字段 | 作用 | 规范性 | 主要参考 | 备注 |
|---|---|---|---|---|---|
| blocks | `description`、`components` | block 身份与行为 | tutorial/reference hybrid | O8-BLOCKS | 可继续递归到 block components |
| items | `description`、`components` | item 身份与行为 | tutorial/reference hybrid | O8-ITEMS | 可继续递归到 item components |
| loot_tables | `pools`、`entries`、`functions`、`conditions` | 掉落表逻辑 | tutorial/reference hybrid | O8-LOOTTRADE | 适合后续拆成独立页 |
| trading | `tiers`、`groups`、`trades` | 交易表逻辑 | tutorial/reference hybrid | O8-LOOTTRADE | 适合后续拆成独立页 |
| spawn_rules | `description`、`conditions`、各 `minecraft:*` filter/component | 生成规则逻辑 | reference/example hybrid | O8-SPAWN | 适合后续拆成独立页 |

## 5. 写作边界

1. 这页是行为包“表类资源”伞页，不代表这些族以后不再拆分。
2. 当前最值得继续独立拆开的通常是：
   - `loot_tables`
   - `trading`
   - `spawn_rules`
3. `structures` 先只保留为结构层说明，不往字段层扩，因为它本身不是 JSON 文件族。
4. 真正的字段级拆分已经转移到递归索引页；本页继续承担“伞页 / 导航页”职责，而不是和递归索引重复抄写整棵树。
