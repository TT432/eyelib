# Bedrock Add-On 来源清单

## 使用规则

本目录里的引用分成五类：

1. **官方机器可读 schema**：Mojang 官方 schema 仓库、catalog、索引与 schema 文件。
2. **官方文档参考**：Microsoft Learn / Minecraft Creator 的稳定页、schema 页、validation 页。
3. **社区解释**：高可信社区文档，用来解释工作流和边界。
4. **真实样例**：官方或社区仓库，用来证明“现实中确实这样写过”，不直接等同于规范。
5. **仓内 importer / fixture 证据**：本仓库 importer、codec、fixture、测试；用于说明 Eyelib 当前实际识别或未识别到什么，不等同于官方规范。

## 七、使用建议

### 1. 讨论字段作用时

优先顺序应是：

1. 官方机器可读 schema（S*）
2. 字段专页（如 `header` / `module` / `metadata` / `dependency`）
3. 总 manifest 页
4. 官方示例页
5. 社区解释
6. 真实样例
7. 仓内 importer 证据（I*，仅说明 Eyelib 当前支持面）

### 2. 讨论结构时

优先顺序应是：

1. 官方工作流页 / RP / BP 入门页
2. 官方 schema / validation 中能落到路径或对象名的条目
3. 官方样本库目录树
4. 社区文件类型和项目结构页
5. 真实样例仓库
6. 仓内 importer 分类证据（I1 / I2）

### 3. 讨论边界案例时

例如：

- `pbr`
- 某些旧字段名
- 某些样例里出现但官方未明确定义的路径

应优先写成：

> 样例存在 / 社区使用 / 项目可管理，但未必是官方稳定标准入口。
