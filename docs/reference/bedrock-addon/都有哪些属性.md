# Bedrock Add-On 都有哪些属性

## 1. 使用范围

本页把 Bedrock Add-On / Pack / `manifest.json` 层面的字段整理成**逐字段参考矩阵**。

每一行尽量回答 8 件事：

1. 字段名是什么。
2. 它属于哪一层。
3. 它的作用是什么。
4. 适用于什么 pack / module。
5. 是必选还是可选。
6. 它属于 stable、preview，还是 sample/community-only 语义。
7. 官方参考应该看哪一页。
8. 当前文档写它时需要注意什么边界。

> 注意：本页只讨论 Add-On / Pack / manifest 层面的字段，不讨论 entity、particle、render controller、recipe 等业务 JSON 内部字段。

## 2. 读表规则

- **层级**：字段属于哪个对象或层，如顶层、`header`、`modules[]`。
- **适用范围**：该字段在哪类 pack / module 下有意义。
- **状态**：
  - `stable`：当前稳定官方参考中明确存在。
  - `preview`：官方 preview / schema 文档中存在。
  - `sample-only`：样例/社区里存在，但不应直接上升为稳定规范。
- **参考编号**：见 `来源清单.md`。

## 3. 容器与分发层字段矩阵

这一节不是 `manifest.json` 字段，而是 Add-On 在导入/分发层面的“外部属性”。保留这一节是为了避免把容器和 manifest 顶层字段混淆。

| 字段/维度 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| 容器类型 | Add-On 分发层 | 决定内容如何导入和打包 | 文件夹 / `.mcpack` / `.mcaddon` | 非 manifest 字段 | stable | O1, C1 | 这是导入容器，不是 manifest 字段 |
| 资产粒度 | Add-On 分发层 | 区分单 pack 与复合导入容器 | 单包 / 多包组合 | 非 manifest 字段 | stable/community clarified | C1 | 社区文档对 `.mcaddon` 组合语义解释更清楚 |
| 安装位置 | 工作流层 | 指导开发态与导入态目录 | RP/BP 开发目录与导入目录 | 非 manifest 字段 | stable | O1, C2 | 属于工作流，不属于 pack 字段 |

## 4. `manifest.json` 顶层字段矩阵

| 字段 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| `format_version` | manifest 顶层 | 指示 manifest 语法版本 | 所有 manifest | 必选 | stable / preview 分叉 | O5-MAIN, O5-VERSION | 文档必须区分 v2、skin v1、preview v3 |
| `header` | manifest 顶层 | 描述 pack 身份与展示信息 | 所有 manifest | 必选 | stable | O5-MAIN, O5-HEADER | 建议引用字段专页而不是只引总页 |
| `modules` | manifest 顶层 | 描述 manifest 包含哪些模块 | 所有 manifest | 必选 | stable | O5-MAIN, O5-MODULE | 需要和 `header` 区分职责 |
| `dependencies` | manifest 顶层 | 描述 pack 或脚本模块依赖 | 需要依赖关系的 pack | 可选 | stable | O5-MAIN, O5-DEP | 应区分 pack 依赖和脚本 API 依赖 |
| `metadata` | manifest 顶层 | 描述作者、许可证、工具信息等 | 任意 pack | 可选 | stable | O5-MAIN, O5-META | 不能把真实样例常用误写成“默认必选” |
| `capabilities` | manifest 顶层 | 声明额外能力开关 | 任意 pack | 可选 | stable | O5-MAIN, O5-CAP | `pbr` 不能直接并入稳定主表能力 |
| `settings` | manifest 顶层 | 定义 pack 设置 UI 项 | 使用 manifest v3 / preview 的 pack | 可选 | preview | O5-MAIN, O5-SETTINGS | 必须明确 preview/experimental 边界 |
| `subpacks` | manifest 顶层 | 定义子包结构 | 使用 preview schema 的 pack | 可选 | preview | O5-SCHEMA, O5-SUBPACK | 当前稳定字段页通常不会把它当主线字段 |
| `has_education_metadata` | manifest 顶层 | Education 相关标记 | preview schema 语境 | 可选 | preview | O5-SCHEMA | 官方有 schema，但说明较弱 |
| `pack_optimization_version` | manifest 顶层 | 与 v3 优化相关 | preview schema 语境 | 可选 | preview | O5-SCHEMA | 当前应按 preview 字段处理 |
| `platform_locked` | manifest 顶层 | 平台锁定相关字段 | preview schema 语境 | 可选 | preview | O5-SCHEMA | 当前应按 preview 字段处理 |

## 5. `header` 字段矩阵

| 字段 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| `name` | `header` | pack 名称 | 所有 pack | 官方页给出语义，但 required 标注不总统一 | stable | O5-HEADER | 示例中几乎总是出现 |
| `description` | `header` | pack 描述 | 所有 pack | 官方页给出语义，但 required 标注不总统一 | stable | O5-HEADER | 常用于 UI 展示 |
| `uuid` | `header` | pack 唯一标识 | 所有 pack | 核心字段 | stable / preview 语义不同 | O5-HEADER | 稳定文档通常表现为字符串；v3 schema 语境需单独说明 |
| `version` | `header` | pack 版本 | 所有 pack | 核心字段 | stable / preview 语义不同 | O5-HEADER, O5-VERSION | v2 常见数组，v3 语境可能是字符串 |
| `min_engine_version` | `header` | 最低兼容游戏版本 | BP / RP | 核心字段 | stable | O5-HEADER, O5-LATEST | 应补版本策略说明 |
| `pack_scope` | `header` | 资源包作用域 | **仅 Resource Pack** | 可选 | stable | O5-HEADER, O5-CHKMANIF | 文档必须显式标 RP-only |
| `base_game_version` | `header` | 模板相关基础版本 | **World Template** | 可选 | stable | O5-HEADER | 不能泛化为普通 BP/RP 字段 |
| `allow_random_seed` | `header` | 随机种子控制 | **World Template** | 可选 | stable | O5-HEADER | 不能泛化为普通 BP/RP 字段 |
| `lock_template_options` | `header` | 锁定模板选项 | **World Template** | 可选 | stable | O5-HEADER | 不能泛化为普通 BP/RP 字段 |

## 6. `modules[]` 字段矩阵

| 字段 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| `description` | `modules[]` | 模块描述 | 任意模块 | 语义存在，但 required 标注不总统一 | stable | O5-MODULE | 不要把“示例里常见”直接写成“显式必选” |
| `type` | `modules[]` | 模块类型 | 任意模块 | 核心字段 | stable | O5-MODULE | 稳定主表优先写 `resources`、`data`、`script`、`world_template` |
| `uuid` | `modules[]` | 模块 UUID | 任意模块 | 核心字段 | stable | O5-MODULE | 与 `header.uuid` 不是同一字段 |
| `version` | `modules[]` | 模块版本 | 任意模块 | 核心字段 | stable | O5-MODULE | 需要与 pack 版本分开说明 |
| `language` | `modules[]` | 脚本语言 | **script module only** | 可选（仅脚本模块语义中有意义） | stable | O5-MODULE | 不能泛化到所有模块 |
| `entry` | `modules[]` | 脚本入口文件 | **script module only** | 可选（仅脚本模块语义中有意义） | stable | O5-MODULE | 不能泛化到所有模块 |
| `client_data` | 示例历史项 | 在官方示例中出现过的旧/历史语境项 | 历史示例语境 | 不建议作为主表字段写入 | sample-only / legacy | O6 | 如果提到，必须标明“示例/历史上下文，不是当前稳定主表字段” |

## 7. `dependencies[]` 字段矩阵

| 字段 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| `uuid` | `dependencies[]` | 指向另一个 pack 的 UUID | pack-to-pack 依赖 | 可选 | stable | O5-DEP | 文档应写明是 pack 依赖主形式 |
| `module_name` | `dependencies[]` | 指向脚本模块名 | scripting/API 依赖 | 可选 | stable | O5-DEP | 不应写成普通 BP↔RP 主依赖方式 |
| `version` | `dependencies[]` | 依赖版本 | pack 或 API 依赖 | 可选 | stable | O5-DEP | 应说明 pack version 与 API semver 的差异 |

| 字段 | 层级 | 作用 | 适用范围 | 必选/可选 | 状态 | 主要参考 | 备注 |
|---|---|---|---|---|---|---|---|
| `authors` | `metadata` | 作者列表 | 任意 pack | 稳定语义下可选 | stable | O5-META | 如果覆盖 v3 preview，应补“author-required 已知问题” |

| `product_type` | `metadata` | 产品类型 | 任意 pack | 可选 | stable | O5-META | 当前应收敛到 `addon` 语义，不要泛化 |
| `url` | `metadata` | 项目/仓库地址 | 任意 pack | 可选 | stable | O5-META | 常见于真实样例 |

|---|---|---|---|---|---|---|---|

| `editorExtension` | `capabilities[]` | 与编辑器扩展相关 | 有相关需求的 pack | 可选 | stable | O5-CAP | 官方主表能力 |
| `experimental_custom_ui` | `capabilities[]` | 启用实验性自定义 UI | 有相关需求的 pack | 可选 | stable | O5-CAP | 官方主表能力 |
| `raytraced` | `capabilities[]` | 启用 ray tracing 相关能力 | 有相关需求的 pack | 可选 | stable | O5-CAP | 官方主表能力 |

## 12. 当前文档写作准则

后续继续写 `docs/reference/bedrock-addon` 时，任何字段都应先判断它属于哪一类：

1. **stable**：可以当成当前稳定官方字段来写。
2. **preview**：可以写，但必须明确 preview / experimental 边界。
3. **sample-only / legacy**：只能写成“样例或旧语境中出现过”，不能偷换成当前稳定规范。
