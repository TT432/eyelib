# ST-E: ArmorLayer mixin 实现盔甲 attachable 渲染

## 前置条件
- ST-A/B/C/D 已完成
- `AttachableItemRenderSetup.getOrPrepare(LivingEntity, EquipmentSlot, boolean)` 可用
- `AttachableResolver.resolve(holder, stack)` 已支持 item molang 条件评估
- 缓存 key 为 `EquipmentSlot`，支持盔甲槽

## 目标
Mixin `HumanoidArmorLayer.renderArmorPiece`，按 EquipmentSlot 独立拦截盔甲渲染：
- 有 attachable 定义 → 渲染 attachable 模型，cancel vanilla 盔甲渲染
- 无 attachable 定义 → vanilla 正常渲染

## 后置条件
- 盔甲槽（HEAD/CHEST/LEGS/FEET）物品有 attachable 定义时，attachable 模型替代 vanilla 盔甲模型
- attachable 动画通过 `tickForEntity` 正常 tick（已扩展到 6 个 EquipmentSlot）
- `context.item_slot` 设置为 `slot.armor.head/chest/legs/feet`
- `context.is_first_person` 设置（HumanoidArmorLayer 只在第三人称渲染，本地玩家 F5 第一人称不显示身体）

## 不变量
- 每个 EquipmentSlot 独立判断和 cancel（不影响其他槽位的 vanilla 渲染）
- attachable 模型渲染在 holder model 对应骨骼位置（head/body/rightLeg）
- 通过 `getContextModel()` 获取 holder model（有最新动画），而非 armor model 参数（HEAD 注入时未 copyPropertiesTo）

## 异常行为
- `AttachableResolver.resolve` 返回 null → 不 cancel，vanilla 渲染
- `getOrPrepare` 返回 null（attachable 加载失败）→ 不 cancel，vanilla 渲染
- `getContextModel()` 返回非 HumanoidModel → 不应发生（HumanoidArmorLayer 泛型约束）

## 副作用
- `getOrPrepare` 会触发 RenderData lazy-init 和 parent_setup（ST-C）
- mixin 注册到 `eyelib.mixins.json` client 列表（`//? if !modern` 分支，仅 1.20.1）

## 设计决策

### 1. Mixin renderArmorPiece（private 方法）而非 render
`render` 是整体方法，cancel 会影响所有 4 个盔甲槽。`renderArmorPiece` 按 slot 独立调用，HEAD 注入 cancellable 可精确控制每个槽位。

### 2. 用 getContextModel() 而非 model 参数
`renderArmorPiece` 的 `model` 参数（armor model）在 HEAD 注入时还没被 `copyPropertiesTo` 同步动画。用 `getContextModel()`（holder model）获取最新动画状态的 ModelPart。

### 3. EquipmentSlot → ModelPart 映射
- HEAD → `model.head`
- CHEST → `model.body`
- LEGS → `model.rightLeg`
- FEET → `model.rightLeg`

LEGS/FEET 都映射到 rightLeg，因为 attachable 模型通常包含双腿结构，起始位置用右腿（左腿对称）。BE attachable 的 bone binding 机制会进一步处理精确的骨骼跟随。

### 4. isLocalPlayerFirstPerson 传给 getOrPrepare
虽然 HumanoidArmorLayer 只在第三人称渲染，但仍传 `isLocalPlayerFirstPerson(entity)` 以保证 context 正确（与手持路径一致）。

### 5. tickForEntity 扩展到 6 个 EquipmentSlot
从 `{MAINHAND, OFFHAND}` 扩展到 `{MAINHAND, OFFHAND, HEAD, CHEST, LEGS, FEET}`，让盔甲槽 attachable 也能 tick 动画。

## 实施步骤
1. 创建 `HumanoidArmorLayerMixin.java`（mixin/client 包）
2. `@Shadow getContextModel()` 访问 holder model
3. `@Inject renderArmorPiece HEAD cancellable`，解析 attachable，渲染 + cancel
4. `AttachableItemRenderSetup.isLocalPlayerFirstPerson` 改为 public（mixin 跨包访问）
5. `tickForEntity` 扩展遍历 6 个 EquipmentSlot
6. 注册 mixin 到 `eyelib.mixins.json` client 列表
