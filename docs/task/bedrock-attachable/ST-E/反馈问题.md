# ST-E 反馈问题

## 1. HumanoidArmorLayerMixin @Shadow 继承方法导致客户端崩溃

### 现象
启动 1.20.1 客户端即崩溃：
```
@Shadow method getParentModel was not located in target class HumanoidArmorLayer. No refMap loaded.
```

### 根因
- ST-E 初版用 `@Shadow getParentModel()` 访问 holder model（用于同步骨骼动画）
- `getParentModel()` 是 **RenderLayer 的方法**，HumanoidArmorLayer 通过继承获得
- **Mixin AP 0.8.5 不为继承方法的 @Shadow 生成 refmap 条目**
  - refmap.json 中 HumanoidArmorLayerMixin 只有 `renderArmorPiece`（@Inject target），没有 `getParentModel`
  - 运行时 class 是 SRG 名（`m_7486_`），无 refmap 映射则按字面名 `getParentModel` 匹配失败 → 崩溃

### 验证方法
1. 检查 `versions/1.20.1/build/resources/main/eyelib.refmap.json`（或 `build/mixin/eyelib.refmap.json`）是否有该 @Shadow 的映射条目
2. 对照 MC 源码（`.local_ref/mc/1.20.1/sources`）确认方法是继承还是目标类自身声明

### 修复（commit 98410206）
- 去掉 `@Shadow getParentModel()`
- 改用 `@At(value="INVOKE", target="Lnet/minecraft/client/model/HumanoidModel;copyPropertiesTo(...)V", shift=AFTER)` 锚点
- 在该锚点之后，方法参数 `model`（p_model）已被 `copyPropertiesTo` 同步完所有骨骼动画，可直接使用
- **@At INVOKE target 会生成 refmap 条目**（不同于 @Shadow 继承方法），refmap 验证通过

### 通用教训
- **避免 @Shadow 目标类的继承方法**（Mixin AP 0.8.5 不生成 refmap 条目）
- 改用 @At INVOKE 锚点（注入到方法调用之后取局部变量/参数），或 @Accessor（仅对目标类自身声明的字段/方法）
- mixin-writing skill 的"常见坑 #1"已有 refmap 验证方法，但未明确"继承方法不生成条目"这一点，建议后续补充
