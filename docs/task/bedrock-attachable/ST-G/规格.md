## 设计决策

### 不新增 Molang context 值断言
clientsmoke 验证"接线链路是否工作"（黑盒），不验证内部状态。is_first_person/parent_setup 的值正确性由单测覆盖（ST-B/C 各自有测试）。clientsmoke 通过 getOrPrepare 不抛异常间接验证这些逻辑路径被正确执行。

### test_helmet 复用 test_stick 资源
- geometry: `geometry.eyelib.test_stick`（root bone binding 用 `q.item_slot_to_bone_name`）
- render_controller: `controller.render.eyelib.test_stick`
- material: `entity_alphatest`
- texture: `minecraft:textures/item/stick`
- 只改 identifier (`eyelib:test_helmet`) 和 item (`minecraft:iron_helmet`)
- 目的：最小新增资源，验证 resolve + EquipmentSlot 缓存路径

### 装备到玩家盔甲槽的方式
用 `player.setItemSlot(EquipmentSlot.HEAD, helmet)`，这是 vanilla LivingEntity API。
