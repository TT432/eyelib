# Minecraft 26.1.2 几何渲染管线

> 基于原版源码（`net/minecraft` + `com/mojang/blaze3d`）分析，NeoForge 扩展部分标注。

## 概述

26.1.2 的渲染架构经过了一次**重大重构**，核心设计理念从"OpenGL 状态机 + 立即模式绘制"转向了**声明式渲染管线（Render Pipeline State Object）+ 帧图（Frame Graph）+ 提交节点（Submit Node）**的现代 GPU API 范式。以下是关键架构变化：

### 核心架构变化

| 概念 | 旧范式（1.20.x 等） | 26.1.2 新范式 |
|------|---------------------|---------------|
| GPU 管线状态 | 代码中手动 `RenderSystem.glEnable/glBlendFunc` | **`RenderPipeline`** 声明式 PSO（Pipeline State Object） |
| 着色器管理 | 每个 RenderType 绑定一个 `ShaderInstance` | **`ShaderManager`** 集中管理源码，管线预编译 |
| 渲染类型 | `RenderType` 内含所有状态 | **`RenderType` = name + `RenderSetup`**（包装 `RenderPipeline` + 运行时纹理/目标） |
| 实体渲染 | 立即模式：`render()` 直接写顶点并绘制 | **两阶段**：先 `submit()` 收集提交节点，后由 FeatureRenderer 批量绘制 |
| 实体数据 | 渲染器直接持有 Entity | **`EntityRenderState`**：先 `extractRenderState()` 提取，再 `submit()` |
| 区块网格 | `ChunkRenderDispatcher` + `BufferBuilder` | **`SectionCompiler`** + **`CompiledSectionMesh`** + **`UberGpuBuffer`** |
| 区块缓冲 | 每个 RenderSection 一个 `VertexBuffer` | **全局 UberGpuBuffer** + TLSF 分配器，按 section 切片 |
| 渲染调度 | 直接 `draw` / `endBatch` | **`FrameGraphBuilder`** 声明渲染 Pass，统一执行 |
| 光照计算 | `ModelBlockRenderer` 内联 | **`BlockModelLighter`** 独立负责 AO/Flat 光照 |

### 两条核心管线

```
方块：BlockState → BlockStateModelDispatcher → BlockStateModel(烘焙) 
     → ModelBlockRenderer + BlockModelLighter → BakedQuad → BufferBuilder 
     → MeshData → UberGpuBuffer(GPU) → RenderPass.drawMultipleIndexed

实体：Entity → extractRenderState() → EntityRenderState 
     → EntityRenderer.submit() → SubmitNodeStorage 
     → FeatureRenderDispatcher.renderXxxFeatures() 
     → ModelFeatureRenderer → ModelPart.compile() → VertexConsumer(BufferBuilder) 
     → MeshData → RenderType.draw() → RenderPass.drawIndexed
```

## 1. 方块几何管线

### 1.1 模型定义与分发

方块模型的"定义→选择→烘焙"分布在 `block/dispatch/` 和 `block/model/` 两个包中。

#### 1.1.1 模型分发器：`BlockStateModelDispatcher`

这是一个 **record**，描述如何根据 BlockState 选择模型：

```java
// block/dispatch/BlockStateModelDispatcher.java
public record BlockStateModelDispatcher(
    Optional<SimpleModelSelectors> simpleModels,  // "variants" 定义
    Optional<MultiPartDefinition> multiPart        // "multipart" 定义
) {
    public Map<BlockState, BlockStateModel.UnbakedRoot> instantiate(
        StateDefinition<Block, BlockState> stateDefinition, Supplier<String> source
    ) { ... }
}
```

- 解析 `assets/<modid>/blockstates/<block>.json` 中的 `variants` 或 `multipart` 字段
- `SimpleModelSelectors`：将 BlockState 的属性组合（如 `"facing=north,powered=true"`）映射到 `BlockStateModel.Unbaked`
- `MultiPartDefinition`：包含一组 `Selector`（条件→模型），`MultiPartModel` 负责根据条件动态组合多个模型部件
- `instantiate()` 输出一个 `Map<BlockState, BlockStateModel.UnbakedRoot>`——每个 BlockState 对应一个未烘焙的模型根

#### 1.1.2 变体（Variant）系统

```java
// block/dispatch/Variant.java
public record Variant(Identifier modelLocation, Variant.SimpleModelState modelState) 
    implements BlockStateModelPart.Unbaked {
    
    public BlockStateModelPart bake(ModelBaker modelBakery) {
        return SimpleModelWrapper.bake(modelBakery, this.modelLocation, this.modelState.asModelState());
    }
}
```

- **`Variant`**：单个变体定义（模型路径 + 旋转状态），实现 `BlockStateModelPart.Unbaked`，`bake()` 产出 `BlockStateModelPart`
- **`SimpleModelState`**：x/y/z 三轴旋转（`Quadrant` 枚举，90° 为单位）+ UV 锁定
- **`WeightedVariants`**：加权变体列表，随机选择一个变体
- **`SingleVariant`**：单变体 `BlockStateModel` 实现
- **`MultiPartModel`**：多部件模型，根据 `Selector` 条件动态收集匹配的 `BlockStateModelPart`

#### 1.1.3 烘焙后的模型

```java
// block/dispatch/BlockStateModel.java
public interface BlockStateModel {
    void collectParts(RandomSource random, List<BlockStateModelPart> output);
    // ...
}
```

- **`BlockStateModel`**：烘焙后的方块状态模型接口，核心方法 `collectParts()` 输出一组 `BlockStateModelPart`
- **`BlockStateModelPart`**：单个模型部件，提供 `getQuads(Direction)` 返回该面的 `BakedQuad` 列表
- **`BakedQuad`**（`resources/model/geometry/BakedQuad.java`）：烘焙后的四边形，是一个 record：

```java
public record BakedQuad(
    Vector3fc position0, position1, position2, position3,  // 4 个顶点位置

    Direction direction,                                     // 朝向
    MaterialInfo materialInfo,                               // 材质信息
    ...

    public record MaterialInfo(
        TextureAtlasSprite sprite, ChunkSectionLayer layer, 
        RenderType itemRenderType, int tintIndex,

```

  > **注意**：`BakedQuad` 移到了 `resources/model/geometry/` 包，且不再是 `int[]` 顶点数据数组——而是直接持有 `Vector3fc` 位置 + packed UV + `MaterialInfo`（包含 sprite、区块层、渲染类型、tint 索引等）。这是一个更面向对象的表示。

#### 1.1.4 模型集合：`BlockStateModelSet`

```java
// block/BlockStateModelSet.java

    private final Map<BlockState, BlockStateModel> modelByState;
    public BlockStateModel get(BlockState state) { ... }
}
```

- 烘焙完成后，所有 BlockState → BlockStateModel 的映射存储在 `BlockStateModelSet` 中
- `SectionCompiler` 通过 `blockModelSet.get(blockState)` 获取每个方块的烘焙模型

### 1.2 模型烘焙（顶点生成）

#### 1.2.1 `ModelBlockRenderer` — 方块网格化入口

`ModelBlockRenderer` 负责遍历模型部件、做面剔除判断、调用光照计算、输出四边形。它在 26.1.2 中仍然存在，但光照逻辑被拆分到了 `BlockModelLighter`。

```java
// block/ModelBlockRenderer.java
public void tesselateBlock(
    BlockQuadOutput output, float x, float y, float z, 
    BlockAndTintGetter level, BlockPos pos, BlockState blockState, 
    BlockStateModel model, long seed
) {
    model.collectParts(level, pos, blockState, this.random, this.parts);
    // 根据 AO 设置选择渲染路径
    if (useAO) {
        this.tesselateAmbientOcclusion(output, x, y, z, parts, level, blockState, pos);
    } else {
        this.tesselateFlat(output, x, y, z, parts, level, blockState, pos);
    }
}
```

**渲染流程**（以 `tesselateAmbientOcclusion` 为例）：
1. `model.collectParts()` → 获取所有 `BlockStateModelPart`
2. 对每个 part，遍历 6 个方向：
   - `shouldRenderFace()` 判断该面是否应渲染（被遮挡则跳过）
   - 对可见面的 quads：根据 quad 的 `ambientOcclusion` 标志选择 AO 或 Flat 光照
3. 遍历无方向 quads（`getQuads(null)`）
4. 对每个 quad 调用 `BlockModelLighter.prepareQuadAmbientOcclusion()` 或 `prepareQuadFlat()` 计算光照
5. `putQuadWithTint()` → `output.put()` 输出到 `BlockQuadOutput`

**`BlockQuadOutput`** 是一个函数式接口：
```java
// block/BlockQuadOutput.java
@FunctionalInterface
public interface BlockQuadOutput {
    void put(float x, float y, float z, BakedQuad quad, QuadInstance instance);
}
```

#### 1.2.2 `BlockModelLighter` — 光照计算器

`BlockModelLighter` 负责**环境光遮蔽（AO）和平面光照（Flat）**的计算，将结果写入 `QuadInstance`。

```java
// block/BlockModelLighter.java
public class BlockModelLighter {
    // AO 光照：计算每个顶点的阴影和光线坐标
    public void prepareQuadAmbientOcclusion(
        BlockAndTintGetter level, BlockState state, BlockPos centerPosition, 
        BakedQuad quad, QuadInstance outputInstance
    ) { ... }
    
    // 平面光照：使用面的方向光照
    public void prepareQuadFlat(
        BlockAndTintGetter level, BlockState state, BlockPos pos, 
        int lightCoords, BakedQuad quad, QuadInstance outputInstance
    ) { ... }
    
    // 计算面的形状信息（是否是完整方块面、是否部分面）
    protected void prepareQuadShape(...) { ... }
}
```

**AO 算法**（`prepareQuadAmbientOcclusion`）：
1. `prepareQuadShape()` 计算面的边界框，判断 `faceCubic`（完整方块面）和 `facePartial`（部分面）
2. 采样面四角和中心相邻方块的光照和遮蔽数据（`Cache` 类缓存 100 条记录）
3. 计算每个顶点的 AO 阴影值（`AdjacencyInfo` + `AmbientVertexRemap`）
4. 对于部分面，使用 `faceShape` 权重做加权混合
5. 应用 `CardinalLighting`（方向光照系数）
6. 结果写入 `QuadInstance`（每顶点 color + lightCoords）

**`QuadInstance`**（`com.mojang.blaze3d.vertex`）：
```java
public class QuadInstance {
    private int color0, color1, color2, color3;       // 每顶点颜色
    private int lightCoords0..3;                        // 每顶点光照坐标
    private int overlayCoords;                          // overlay（方块不用）
    // setColor/setLightCoords/scaleColor...
}
```

#### 1.2.3 `BuiltInBlockModels` — 内置方块模型

`BuiltInBlockModels.java`（26.5KB）负责特殊方块模型的注册和烘焙，是 `BlockStateModel.UnbakedRoot` 的工厂系统。

### 1.3 区块网格化

区块渲染采用**编译→上传→绘制**三步分离架构。

#### 1.3.1 编译：`SectionCompiler`

```java
// chunk/SectionCompiler.java
public SectionCompiler.Results compile(
    SectionPos sectionPos, RenderSectionRegion region, 
    VertexSorting vertexSorting, SectionBufferBuilderPack builders, 
    List<...> additionalRenderers
) {
    BlockModelLighter.enableCaching();
    ModelBlockRenderer blockRenderer = new ModelBlockRenderer(this.ambientOcclusion, true, this.blockColors);
    FluidRenderer fluidRenderer = new FluidRenderer(this.fluidModelSet);
    Map<ChunkSectionLayer, BufferBuilder> startedLayers = new EnumMap<>(ChunkSectionLayer.class);
    
    // BlockQuadOutput 回调：将 quad 写入对应层的 BufferBuilder
    BlockQuadOutput quadOutput = (x, y, z, quad, instance) -> {
        BufferBuilder builder = this.getOrBeginLayer(startedLayers, builders, quad.materialInfo().layer());
        builder.putBlockBakedQuad(x, y, z, quad, instance);
    };
    
    // 遍历 section 内所有方块
    for (BlockPos pos : BlockPos.betweenClosed(minPos, maxPos)) {
        BlockState blockState = region.getBlockState(pos);
        // 流体渲染
        if (!fluidState.isEmpty()) {
            fluidRenderer.tesselate(region, pos, fluidOutput, blockState, fluidState);
        }
        // 方块模型渲染
        if (blockState.getRenderShape() == RenderShape.MODEL) {
            blockRenderer.tesselateBlock(quadOutput, relX, relY, relZ, region, pos, 
                blockState, this.blockModelSet.get(blockState), blockState.getSeed(pos));
        }
    }
    
    // 构建每层的 MeshData
    for (Entry<ChunkSectionLayer, BufferBuilder> entry : startedLayers.entrySet()) {
        MeshData mesh = entry.getValue().build();
        if (mesh != null) {
            results.renderedLayers.put(layer, mesh);
        }
    }
}
```

**关键点**：
- 每个 section（16×16×16）编译时，遍历所有方块
- `quad.materialInfo().layer()` 决定 quad 属于哪个 `ChunkSectionLayer`（SOLID / CUTOUT / TRANSLUCENT）
- 每层使用独立的 `BufferBuilder`，编译后生成 `MeshData`
- `ChunkSectionLayer` 直接绑定 `RenderPipeline`：

```java
// chunk/ChunkSectionLayer.java
public enum ChunkSectionLayer {
    SOLID(RenderPipelines.SOLID_TERRAIN, 4194304, false),
    CUTOUT(RenderPipelines.CUTOUT_TERRAIN, 4194304, false),
    TRANSLUCENT(RenderPipelines.TRANSLUCENT_TERRAIN, 786432, true);
}
```

#### 1.3.3 GPU 上传：UberGpuBuffer

26.1.2 不再为每个 RenderSection 创建独立的 GPU `VertexBuffer`，而是使用**全局大缓冲（Uber Buffer）+ TLSF 内存分配器**：

```java
// chunk/SectionRenderDispatcher.java
public class SectionRenderDispatcher {
    // 每层一个 UberGpuBuffer（顶点 + 透明层的索引）
    private final Map<ChunkSectionLayer, SectionUberBuffers> chunkUberBuffers;
    
    // 构造：创建 128MB 顶点堆 + 32MB 索引堆
    UberGpuBuffer<SectionMesh> vertexUberBuffer = new UberGpuBuffer<>(
        layer.label(), 32, 134217728, vertexFormat.getVertexSize(), gpuDevice, 33554432, workarounds
    );
}
```

**上传流程**（`RebuildTask.doTask()`）：
1. `SectionCompiler.compile()` → `Results`（含每层 `MeshData`）
2. 创建 `CompiledSectionMesh`
3. 对每层：`RenderSection.addSectionBuffersToUberBuffer()` → `UberGpuBuffer.addAllocation()` 将数据加入暂存区
4. `uploadGlobalGeomBuffersToGPU()` → `UberGpuBuffer.uploadStagedAllocations()` 批量上传到 GPU
5. 上传完成回调 → `checkSectionMesh()` → 当所有层上传完成后，切换 `sectionMesh` 引用

**`UberGpuBuffer`**（`com.mojang.blaze3d.vertex`）使用 **`TlsfAllocator`**（Two-Level Segregated Fit）在大缓冲中分配/释放切片，每个 section 对应一个偏移量。

#### 1.3.4 绘制：`ChunkSectionsToRender`

```java
// chunk/ChunkSectionsToRender.java
public record ChunkSectionsToRender(
    GpuTextureView textureView,                                          // 区块纹理图集
    EnumMap<ChunkSectionLayer, Int2ObjectOpenHashMap<List<Draw<GpuBufferSlice[]>>>> drawGroupsPerLayer,
    int maxIndicesRequired,
    GpuBufferSlice[] chunkSectionInfos                                    // ChunkSection uniform 数据
) {
    public void renderGroup(ChunkSectionLayerGroup group, GpuSampler sampler) {
        try (RenderPass renderPass = RenderSystem.getDevice()
                .createCommandEncoder()
                .createRenderPass(...)) {
            RenderSystem.bindDefaultUniforms(renderPass);
            renderPass.bindTexture("Sampler0", this.textureView, sampler);       // 区块纹理
            renderPass.bindTexture("Sampler2", lightmap, linearSampler);         // 光照图
            
            for (ChunkSectionLayer layer : group.layers()) {
                renderPass.setPipeline(layer.pipeline());                        // 设置 RenderPipeline
                for (List<Draw<GpuBufferSlice[]>> draws : drawGroup.values()) {
                    // 单次调用批量绘制多个 section！
                    renderPass.drawMultipleIndexed(draws, defaultIndexBuffer, ...);
                }
            }
        }
    }
}
```

> **关键性能优化**：`drawMultipleIndexed()` 在单个 `RenderPass` 中批量绘制所有可见 section，使用 `GpuBufferSlice[]` 引用 UberGpuBuffer 中的切片偏移，并通过 `chunkSectionInfos`（`DynamicUniforms.ChunkSectionInfo`）传递每 section 的 modelView 矩阵。这取代了旧版"每 section 一次 draw call"的模式。

## 2. 实体几何管线

### 2.1 模型定义层

#### 2.1.1 `ModelPart` — 实体几何核心

实体几何由 `ModelPart` 树定义（`client/model/geom/ModelPart.java`）：

```java
public final class ModelPart {
    public float x, y, z;           // 相对父节点的位置（像素单位 / 16）
    public float xRot, yRot, zRot;  // 旋转
    public float xScale, yScale, zScale;
    public final List<ModelPart.Cube> cubes;
    public final Map<String, ModelPart> children;
    
    // 渲染：递归遍历子节点
    public void render(PoseStack poseStack, VertexConsumer buffer, 
        int lightCoords, int overlayCoords, int color) {
        poseStack.pushPose();
        this.translateAndRotate(poseStack);           // 应用平移+旋转+缩放
        this.compile(poseStack.last(), buffer, lightCoords, overlayCoords, color);
        for (ModelPart child : this.children.values()) {
            child.render(poseStack, buffer, lightCoords, overlayCoords, color);
        }
        poseStack.popPose();
    }
}
```

#### 2.1.2 `ModelPart.Cube` — 立方体几何

```java
public static class Cube {
    public final Polygon[] polygons;   // 6 个面（可见的）
    
    public Cube(int xTexOffs, int yTexOffs, float minX, float minY, float minZ, 
        float width, float height, float depth, ...) {
        // 根据纹理偏移和尺寸构建 6 个面的多边形
        // 每个面 = 4 个 Vertex + UV 坐标
    }
    
    // 编译：将多边形写入顶点消费者
    public void compile(PoseStack.Pose pose, VertexConsumer builder, 
        int lightCoords, int overlayCoords, int color) {
        for (ModelPart.Polygon polygon : this.polygons) {
            Vector3f normal = pose.transformNormal(polygon.normal, scratch);
            for (ModelPart.Vertex vertex : polygon.vertices) {
                Vector3f pos = matrix.transformPosition(
                    vertex.worldX(), vertex.worldY(), vertex.worldZ(), scratch);
                builder.addVertex(pos.x, pos.y, pos.z, color, 
                    vertex.u, vertex.v, overlayCoords, lightCoords, nx, ny, nz);
            }
        }
    }
}
```

- 每个 Cube 由 6 个 `Polygon` 组成，每个 Polygon 有 4 个 `Vertex`（位置 + UV）
- `Vertex.worldX/Y/Z()` 将像素坐标转换为世界坐标（`/ 16.0F`）
- `compile()` 方法遍历多边形，将顶点变换后写入 `VertexConsumer`

### 2.2 渲染流程

实体渲染采用 **状态提取 → 提交 → 批量渲染** 三步架构。

#### 2.2.1 状态提取：`extractRenderState()`

```java
}
```

> **关键变化**：渲染不再直接操作 `Entity` 对象，而是先提取到不可变的 `EntityRenderState`。这解耦了游戏逻辑与渲染，使渲染线程可以安全地处理状态快照。

#### 2.2.2 提交阶段：`submit()`

```java
// entity/LivingEntityRenderer.java
public void submit(S state, PoseStack poseStack, SubmitNodeCollector submitNodeCollector, CameraRenderState camera) {
    poseStack.pushPose();
    poseStack.scale(scale, scale, scale);
    this.setupRotations(state, poseStack, state.bodyRot, scale);
    poseStack.scale(-1.0F, -1.0F, 1.0F);
    poseStack.translate(0.0F, -1.501F, 0.0F);
    
    RenderType renderType = this.getRenderType(state, isBodyVisible, forceTransparent, state.appearsGlowing());
    if (renderType != null) {
        // 核心：将模型提交到 SubmitNodeCollector
        submitNodeCollector.submitModel(
            this.model, state, poseStack, renderType, 
            state.lightCoords, overlayCoords, tintedColor, null, state.outlineColor, null
        );
    }
    
    // 提交各 RenderLayer（装备层等）
    for (RenderLayer<S, M> layer : this.layers) {
        layer.submit(poseStack, submitNodeCollector, state.lightCoords, state, state.yRot, state.xRot);
    }
    
    poseStack.popPose();
    super.submit(state, poseStack, submitNodeCollector, camera);  // 名牌、拴绳等
}
```

> **注意**：`submit()` 不执行任何实际绘制。它只是将渲染意图（模型 + 状态 + 变换 + RenderType）记录到 `SubmitNodeCollector` 中。

#### 2.2.3 调度入口：`EntityRenderDispatcher.submit()`

```java
// entity/EntityRenderDispatcher.java
public <S extends EntityRenderState> void submit(
    S renderState, CameraRenderState camera, double x, double y, double z, 
    PoseStack poseStack, SubmitNodeCollector submitNodeCollector
) {
    EntityRenderer<?, ? super S> renderer = this.getRenderer(renderState);
    poseStack.pushPose();
    poseStack.translate(relativeX, relativeY, relativeZ);

        submitNodeCollector.submitFlame(...);
    }
    if (!renderState.shadowPieces.isEmpty()) {
        submitNodeCollector.submitShadow(...);
    }
    poseStack.popPose();
}
```

#### 2.2.4 顶点写入

`ModelPart.compile()` 通过 `VertexConsumer.addVertex(x,y,z,color,u,v,overlay,light,nx,ny,nz)` 写入 11 个属性的完整顶点。`BufferBuilder` 的 `fastFormat` 路径为 `DefaultVertexFormat.ENTITY` 格式提供优化的直接内存写入：

```java
// com/mojang/blaze3d/vertex/BufferBuilder.java
public void addVertex(float x, float y, float z, int color, float u, float v, 
    int overlayCoords, int lightCoords, float nx, float ny, float nz) {
    if (this.fastFormat) {   // ENTITY 或 BLOCK 格式的快速路径
        long pointer = this.beginVertex();
        memPutFloat(pointer + 0, x);      // Position
        memPutFloat(pointer + 4, y);
        memPutFloat(pointer + 8, z);
        putRgba(pointer + 12, color);     // Color
        memPutFloat(pointer + 16, u);     // UV0
        memPutFloat(pointer + 20, v);
        putPackedUv(pointer + 24, overlayCoords);  // UV1 (overlay)
        putPackedUv(pointer + 28, lightCoords);    // UV2 (lightmap)
        memPutByte(pointer + 32, normalInt(nx));   // Normal
        memPutByte(pointer + 33, normalInt(ny));
        memPutByte(pointer + 34, normalInt(nz));
    }
}
```

### 2.3 Feature 渲染层

Feature 系统（`client/renderer/feature/`）是 26.1.2 实体渲染的核心调度器，替代了旧版直接在 `LivingEntityRenderer.render()` 中立即绘制的方式。

#### 2.3.1 `FeatureRenderDispatcher` — 总调度

```java
// feature/FeatureRenderDispatcher.java
public class FeatureRenderDispatcher {
    private final SubmitNodeStorage submitNodeStorage;
    private final MultiBufferSource.BufferSource bufferSource;
    
    // 各 FeatureRenderer
    private final ShadowFeatureRenderer shadowFeatureRenderer;
    private final FlameFeatureRenderer flameFeatureRenderer;
    private final ModelFeatureRenderer modelFeatureRenderer;        // 实体模型
    private final ModelPartFeatureRenderer modelPartFeatureRenderer; // 独立 ModelPart

    private final CustomFeatureRenderer customFeatureRenderer;
    private final ParticleFeatureRenderer particleFeatureRenderer;
    
    public void renderSolidFeatures() {
        for (SubmitNodeCollection collection : this.submitNodeStorage.getSubmitsPerOrder().values()) {
            this.modelFeatureRenderer.renderSolid(collection, bufferSource, ...);
            this.modelPartFeatureRenderer.renderSolid(collection, bufferSource, ...);
            this.flameFeatureRenderer.renderSolid(collection, bufferSource, atlasManager);
            this.leashFeatureRenderer.renderSolid(collection, bufferSource);
            this.itemFeatureRenderer.renderSolid(collection, bufferSource, ...);
            this.blockFeatureRenderer.renderSolid(collection, bufferSource, ...);
            this.customFeatureRenderer.renderSolid(collection, bufferSource);
            this.particleFeatureRenderer.renderSolid(collection);
        }
    }
    
    public void renderTranslucentFeatures() {
        for (SubmitNodeCollection collection : ...) {
            this.shadowFeatureRenderer.renderTranslucent(collection, bufferSource);

            // ...
        }
    }

```

> **关键设计**：`renderSolidFeatures()` 和 `renderTranslucentFeatures()` 分离，允许不透明和半透明实体在不同的帧图 Pass 中渲染（不透明写入深度，半透明在深度拷贝后渲染）。

#### 2.3.2 `ModelFeatureRenderer` — 实体模型渲染

```java
// feature/ModelFeatureRenderer.java
public class ModelFeatureRenderer {
    private <S> void renderModel(
        SubmitNodeStorage.ModelSubmit<S> submit, RenderType renderType, 
        VertexConsumer buffer, ...
    ) {
        // 从 submit 节点恢复 PoseStack
        this.poseStack.setIdentity();
        submit.pose().pose().get(this.poseStack.last().pose());
        // 调用 Model.renderToBuffer() 写入顶点
        submit.model().renderToBuffer(
            this.poseStack, buffer, 
            submit.lightCoords(), submit.overlayCoords(), submit.color()
        );
    }
}
```

**流程**：
1. `renderSolid()` 从 `SubmitNodeCollection` 中取出按 RenderType 分组的 `ModelSubmit` 列表
2. 对每个 RenderType，从 `MultiBufferSource` 获取 `BufferBuilder`（`VertexConsumer`）
3. 遍历 submits，调用 `Model.renderToBuffer()` → `ModelPart.render()` → `Cube.compile()` 写入顶点
4. `bufferSource.endBatch()` 时，`RenderType.draw()` 将 `MeshData` 提交到 GPU

#### 2.3.3 旧版 `layers/` 包

`entity/layers/` 包仍然存在，但 `RenderLayer`（注意：不是旧版 `RenderType` 的别名）的接口改为 `submit()`：
```java
// entity/layers/RenderLayer.java

    public abstract void submit(PoseStack poseStack, SubmitNodeCollector submitNodeCollector, 
        int lightCoords, S state, float yRot, float xRot);

```
这些 layer（如 `HumanoidArmorLayer`、`WolfCollarLayer`、`WingsLayer`）在 `LivingEntityRenderer.submit()` 中被调用，将额外的几何提交到 SubmitNodeCollector。

### 2.4 BlockEntity 渲染（简要）

BlockEntity 使用独立的渲染调度：


### 3.1 RenderType 系统

26.1.2 将旧的 `RenderType`（巨型类，包含所有渲染状态）拆分到了 `rendertype/` 包中。

#### 3.2.3 SubmitNode 系统

SubmitNode 是 26.1.2 实体渲染的**延迟提交**机制：

```
OrderedSubmitNodeCollector (接口，定义 submit* 方法)
    ↑ implements
SubmitNodeCollector (接口，添加 order() 方法)
    ↑ implements              ↑ implements
SubmitNodeStorage            SubmitNodeCollection
（顶层容器，按 order 分组）    （单组提交节点集合）
```

**`SubmitNodeStorage`**：
```java
// renderer/SubmitNodeStorage.java
public class SubmitNodeStorage implements SubmitNodeCollector {
    // 按 order 分组的有序映射
    private final Int2ObjectAVLTreeMap<SubmitNodeCollection> submitsPerOrder = new Int2ObjectAVLTreeMap<>();
    
    public SubmitNodeCollection order(int order) {
        return this.submitsPerOrder.computeIfAbsent(order, ignored -> new SubmitNodeCollection(this));
    }
}
```

**`SubmitNodeCollection`** 持有各种提交节点列表：
```java
// renderer/SubmitNodeCollection.java
public class SubmitNodeCollection implements OrderedSubmitNodeCollector {
    private final List<SubmitNodeStorage.ShadowSubmit> shadowSubmits;
    private final List<SubmitNodeStorage.FlameSubmit> flameSubmits;
    private final ModelFeatureRenderer.Storage modelSubmits;           // 按 RenderType 分组的模型提交
    private final ModelPartFeatureRenderer.Storage modelPartSubmits;
    private final List<SubmitNodeStorage.ItemSubmit> itemSubmits;
    private final List<SubmitNodeStorage.BlockModelSubmit> blockModelSubmits;
    private final List<SubmitNodeStorage.LeashSubmit> leashSubmits;
    // ... text, nameTag, particle, custom 等
}
```

每个 `Submit`（如 `ModelSubmit`）是一个不可变快照，记录：
- `PoseStack.Pose` 副本（变换矩阵）
- `Model` 引用 + `EntityRenderState` 引用
- `RenderType`、`lightCoords`、`overlayCoords`、`color`

> **设计意图**：提交阶段只记录"要画什么"，不实际分配 GPU 资源。渲染阶段可以按类型批量处理、按 RenderType 排序、合并 draw call。

#### 3.4.1 `ShaderManager`

`ShaderManager` 取代了旧版的 `ShaderInstance` 加载系统：

```java
}
```

> **设计**：每帧将所有 per-draw 的 transform/section uniform 写入环形缓冲，draw call 通过 `renderPass.setUniform("DynamicTransforms", slice)` 引用对应的切片。这避免了旧版每帧大量 `glUniform` 调用。

**其他 Uniform**：
- `GlobalSettingsUniform`：全局设置（如雾、光照参数）
- `ProjectionMatrixBuffer`：投影矩阵 uniform buffer
- `UniformValue`：通用 uniform 值封装

### 3.5 主渲染循环

#### 3.5.1 渲染状态对象

26.1.2 引入了 `state/` 包，将渲染状态从渲染器中分离：

```java
// state/GameRenderState.java    — 全局渲染状态
// state/WindowRenderState.java  — 窗口/视口状态
// state/OptionsRenderState.java — 选项状态
// state/level/LevelRenderState.java — 关卡渲染状态（含实体状态列表、天气、天空等）
// entity/state/EntityRenderState.java — 每实体渲染状态
```

#### 3.5.2 帧图（Frame Graph）

`LevelRenderer.renderLevel()` 使用 `FrameGraphBuilder` 声明渲染 Pass：

```java
// renderer/LevelRenderer.java
public void renderLevel(...) {
    FrameGraphBuilder frame = new FrameGraphBuilder();
    
    // 导入外部渲染目标
    this.targets.main = frame.importExternal("main", mainRenderTarget);
    // 创建内部渲染目标（半透明、天气、云等）
    this.targets.translucent = frame.createInternal("translucent", screenSizeTargetDescriptor);
    
    // 声明 Pass
    FramePass clearPass = frame.addPass("clear");
    this.addSkyPass(frame, ...);
    this.addMainPass(frame, ...);          // 核心渲染 Pass
    entityOutlineChain.addToFrame(frame, ...);
    this.addCloudsPass(frame, ...);
    this.addWeatherPass(frame, ...);
    transparencyChain.addToFrame(frame, ...);
    
    // 统一执行帧图
    frame.execute(resourceAllocator, inspector);
}
```

#### 3.5.3 `addMainPass` — 核心渲染 Pass

```java
private void addMainPass(...) {
    pass.executes(() -> {
        // 1. 不透明地形
        chunkSectionsToRender.renderGroup(ChunkSectionLayerGroup.OPAQUE, chunkLayerSampler);
        
        // 2. 提交实体/方块实体/粒子的渲染节点
        this.submitEntities(poseStack, levelRenderState, this.submitNodeStorage);
        this.submitBlockEntities(poseStack, levelRenderState, this.submitNodeStorage);
        levelRenderState.particlesRenderState.submit(this.submitNodeStorage, camera);
        
        // 3. 渲染不透明 Feature（实体模型、物品、方块模型等）
        this.featureRenderDispatcher.renderSolidFeatures();
        bufferSource.endBatch();    // 实际执行所有不透明 draw call
        
        // 4. 复制深度到半透明目标
        translucentTarget.copyDepthFrom(mainTarget);
        
        // 5. 渲染半透明 Feature
        this.featureRenderDispatcher.renderTranslucentFeatures();
        bufferSource.endBatch();
        
        // 6. 半透明地形
        chunkSectionsToRender.renderGroup(ChunkSectionLayerGroup.TRANSLUCENT, chunkLayerSampler);
        
        // 7. 半透明粒子
        this.featureRenderDispatcher.renderTranslucentParticles();
        
        // 8. 清理提交节点
        this.featureRenderDispatcher.clearSubmitNodes();
    });
}
```

> **渲染顺序**：不透明地形 → 不透明实体/Feature → 半透明实体/Feature → 半透明地形 → 半透明粒子。深度在半透明阶段前从主目标复制到半透明目标。

#### 3.5.4 `MultiBufferSource.BufferSource` — 批量绘制

```java
// renderer/MultiBufferSource.java
public static class BufferSource implements MultiBufferSource {
    public VertexConsumer getBuffer(RenderType renderType) {
        // 按 RenderType 获取/创建 BufferBuilder
        // 相同 RenderType 的几何会被合并到同一个 BufferBuilder
    }
    
    private void endBatch(RenderType type, BufferBuilder builder) {
        MeshData mesh = builder.build();
        if (type.sortOnUpload()) {
            mesh.sortQuads(buffer, projectionType.vertexSorting());  // 半透明排序
        }
        type.draw(mesh);   // 提交到 GPU
    }
}
```

### 3.6 GPU 交互

#### 3.6.1 GPU 设备模型

26.1.2 引入了显式的 GPU 设备抽象（类似 WebGPU/Vulkan）：

```
GpuDevice          — GPU 设备（创建缓冲、纹理、管线）
  ↳ CommandEncoder  — 命令编码器（创建 RenderPass、上传缓冲）
      ↳ RenderPass  — 渲染 Pass（设置管线、绑定纹理、绘制）
```

**绘制调用**（`RenderType.draw()`）：
```java
// rendertype/RenderType.java
public void draw(MeshData mesh) {
    // 1. 写入 DynamicTransforms uniform
    GpuBufferSlice dynamicTransforms = RenderSystem.getDynamicUniforms()
        .writeTransform(modelViewMatrix, colorModulator, offset, textureMatrix);
    
    // 2. 上传顶点/索引缓冲到 GPU（immediate 路径）
    GpuBuffer vertices = format.uploadImmediateVertexBuffer(mesh.vertexBuffer());
    GpuBuffer indices = format.uploadImmediateIndexBuffer(mesh.indexBuffer());
    
    // 3. 创建 RenderPass 并绘制
    try (RenderPass renderPass = RenderSystem.getDevice()
            .createCommandEncoder()
            .createRenderPass(name, colorTexture, empty, depthTexture, empty)) {
        renderPass.setPipeline(this.state.pipeline);              // 绑定 RenderPipeline
        RenderSystem.bindDefaultUniforms(renderPass);              // 绑定全局 uniform
        renderPass.setUniform("DynamicTransforms", dynamicTransforms);
        renderPass.setVertexBuffer(0, vertices);
        for (Entry<String, TextureAndSampler> entry : textures.entrySet()) {
            renderPass.bindTexture(entry.getKey(), textureView, sampler);
        }
        renderPass.setIndexBuffer(indices, indexType);
        renderPass.drawIndexed(0, 0, indexCount, 1);
    }
}
```

#### 3.6.2 区块 GPU 路径

区块使用不同的上传策略（非 immediate）：

```
编译线程：
  SectionCompiler.compile() → MeshData
  → UberGpuBuffer.addAllocation() （加入暂存区）

渲染线程：
  uploadGlobalGeomBuffersToGPU()
  → UberGpuBuffer.uploadStagedAllocations() （批量上传到 GPU 大缓冲）

绘制：
  ChunkSectionsToRender.renderGroup()
  → RenderPass.drawMultipleIndexed() （一次调用绘制所有 section）
```

#### 3.6.3 关键 GPU 类

| 类 | 作用 |
|---|---|
| `GpuDevice` | GPU 设备接口，创建所有 GPU 资源 |
| `CommandEncoder` | 命令编码器，录制渲染命令 |
| `RenderPass` | 渲染 Pass，类似 WebGPU 的 GPURenderPassEncoder |
| `GpuBuffer` / `GpuBufferSlice` | GPU 缓冲 / 缓冲切片 |
| `UberGpuBuffer` | 大缓冲管理器（TLSF 分配） |
| `GpuTextureView` | 纹理视图 |
| `GpuSampler` | 纹理采样器 |
| `CompiledRenderPipeline` | 编译后的管线（预编译结果） |
| `RenderTarget` | 渲染目标（颜色+深度纹理） |

## 4. 数据流总结

### 4.1 方块几何完整数据流

```
资源加载阶段（一次性）：
  blockstates/*.json 
    → BlockStateModelDispatcher (Codec 解析)
    → BlockStateModel.UnbakedRoot
    → ModelBaker.bake() (烘焙)
    → BlockStateModel (持有 BlockStateModelPart)
    → BlockStateModelSet (Map<BlockState, BlockStateModel>)

区块编译阶段（异步线程）：
  SectionCompiler.compile(sectionPos)
    ├─ 遍历 16×16×16 方块
    ├─ BlockStateModelSet.get(blockState) → BlockStateModel
    ├─ BlockStateModel.collectParts() → List<BlockStateModelPart>
    ├─ ModelBlockRenderer.tesselateBlock()
    │   ├─ shouldRenderFace() 面剔除
    │   ├─ BlockModelLighter.prepareQuadAmbientOcclusion/Flat() → QuadInstance
    │   └─ BlockQuadOutput.put(x, y, z, BakedQuad, QuadInstance)
    │       └─ BufferBuilder.putBlockBakedQuad() → 写入顶点
    ├─ FluidRenderer.tesselate() (流体)
    └─ BufferBuilder.build() → MeshData (每 ChunkSectionLayer)
  
  → CompiledSectionMesh
  → UberGpuBuffer.addAllocation() (暂存)
  → uploadGlobalGeomBuffersToGPU() (上传到 GPU)

帧渲染阶段：
  ChunkSectionsToRender.renderGroup(OPAQUE/TRANSLUCENT)
    → RenderPass.drawMultipleIndexed() (批量绘制)
```

### 4.2 实体几何完整数据流

```
状态提取阶段（主线程）：
  Entity → EntityRenderer.extractRenderState() → EntityRenderState
    → levelRenderState.entityRenderStates (列表)

提交阶段（渲染线程）：
  EntityRenderDispatcher.submit(renderState, ...)
    → EntityRenderer.submit(state, poseStack, submitNodeCollector)
      → LivingEntityRenderer.submit()
        ├─ submitNodeCollector.submitModel(model, state, pose, renderType, ...)
        ├─ RenderLayer.submit() (装备层等)
        │   └─ submitNodeCollector.submitItem/submitModelPart(...)
        └─ submitNodeCollector.submitLeash/submitNameTag/submitShadow(...)
    → submitNodeStorage (收集所有 SubmitNode)

渲染阶段（FeatureRenderDispatcher）：
  featureRenderDispatcher.renderSolidFeatures()
    ├─ modelFeatureRenderer.renderSolid()
    │   └─ 按 RenderType 分组 → Model.renderToBuffer()
    │       └─ ModelPart.render() → Cube.compile()
    │           └─ VertexConsumer.addVertex(pos, color, uv, overlay, light, normal)
    │               └─ BufferBuilder → MeshData
    ├─ itemFeatureRenderer.renderSolid() (物品)
    ├─ blockFeatureRenderer.renderSolid() (方块模型)
    └─ ...其他 FeatureRenderer
  
  bufferSource.endBatch()
    → RenderType.draw(MeshData)
      → RenderPass.drawIndexed() (提交到 GPU)
```
