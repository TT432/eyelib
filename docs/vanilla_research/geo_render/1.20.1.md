# Minecraft 1.20.1 几何渲染管线

> 本报告基于 Minecraft 1.20.1 (Forge) 原版源码（`net/minecraft` + `com/mojang/blaze3d`）分析，
> 聚焦"几何定义 → 屏幕输出"这条链路。所有类名、方法名、字段均来自源码实际内容。
> 分析范围限定在原版渲染管线，不含 Forge/模组扩展逻辑（涉及处会注明 `[Forge 改动]`）。

## 概述

Minecraft 的几何渲染分为两条相对独立的管线，最终汇入同一套 GPU 上传/绘制基础设施：

- **方块管线**：JSON 模型文件 → Gson 反序列化为 `BlockModel`（含 `BlockElement`/`BlockElementFace`/`BlockFaceUV`）
  → `FaceBakery` 把每个面烘焙成 `BakedQuad`（一个 `int[32]` 数组，封装 position/color/uv/normal）
  → 组装成 `BakedModel`。区块重建时 `ChunkRenderDispatcher.RebuildTask` 遍历每个方块，
  按 `RenderType`（solid/cutout/cutoutMipped/translucent/tripwire）分发到各自的 `BufferBuilder`，
  通过 `ModelBlockRenderer.tesselateBlock` 把 `BakedQuad` 经 `VertexConsumer.putBulkData` 写入，
  最终 `end()` 成 `RenderedBuffer` 并 `VertexBuffer.upload` 上传到 GPU。绘制时 `LevelRenderer.renderChunkLayer`
  对每个区块的 `VertexBuffer` 执行 `draw()`（`glDrawElements`）。

- **实体管线**：`LayerDefinitions.createRoots()` 用 `MeshDefinition`/`PartDefinition`/`CubeDefinition`
  程序化构建模型层级 → `PartDefinition.bake` 烘焙成 `ModelPart` 树（`ModelPart.Cube` 内含预计算的 `Polygon`）。
  渲染时 `LivingEntityRenderer.render` 设置 `PoseStack`（旋转/缩放/平移）→ 调用 `model.setupAnim`
  摆姿势 → `model.renderToBuffer` → `ModelPart.render` 递归遍历 → `Cube.compile` 把每个多边形顶点
  通过 `vertexConsumer.vertex(...)`（完整的 14 参数实体顶点）写入 `VertexConsumer`。叠加的 `RenderLayer`
  （如盔甲、鞘翅）随后依次渲染。数据先入 `MultiBufferSource.BufferSource`，再由 `endBatch` 触发实际绘制。

两者共享：`RenderType`（封装 GL 状态 + shader + texture）、`VertexFormat`（顶点内存布局）、
`BufferBuilder`（CPU 侧顶点缓冲）、`VertexBuffer`（GPU VBO）、`ShaderInstance`（着色器与 Uniform）。

## 1. 方块几何管线

### 1.1 模型定义层

方块模型是 JSON 文件（如 `assets/minecraft/models/block/stone.json`）。反序列化入口是 `BlockModel`：

- **`BlockModel`**（`net/minecraft/client/renderer/block/model/BlockModel.java`，`implements UnbakedModel`）
  - 静态 `GSON` 实例注册了所有反序列化器：
    ```java
    GSON = new GsonBuilder()
        .registerTypeAdapter(BlockModel.class, new BlockModel.Deserializer())
        .registerTypeAdapter(BlockElement.class, new BlockElement.Deserializer())
        .registerTypeAdapter(BlockElementFace.class, new BlockElementFace.Deserializer())
        .registerTypeAdapter(BlockFaceUV.class, new BlockFaceUV.Deserializer())
        .registerTypeAdapter(ItemTransform.class, new ItemTransform.Deserializer())
        .registerTypeAdapter(ItemTransforms.class, new ItemTransforms.Deserializer())
        .registerTypeAdapter(ItemOverride.class, new ItemOverride.Deserializer())
        .create();
    ```
  - 关键字段：`List<BlockElement> elements`（几何）、`Map<String, Either<Material,String>> textureMap`（纹理引用表，`#` 前缀表示引用）、`BlockModel parent`（父模型，支持继承）、`Boolean hasAmbientOcclusion`、`GuiLight guiLight`、`ItemTransforms transforms`（第一/第三人称等显示变换）、`List<ItemOverride> overrides`。
  - `fromStream(Reader)`/`fromString` 从 JSON 解析。`resolveParents(Function)` 解析父模型链。`getElements()` 在自身为空时回退到 parent。

- **`BlockElement`**（描述一个长方体元素）
  - 字段：`Vector3f from`、`Vector3f to`（元素范围，单位 1/16 块，范围限定 `-16.0 ~ 32.0`）、`Map<Direction, BlockElementFace> faces`（每个方向一个面）、`BlockElementRotation rotation`（元素局部旋转：origin/axis/angle/rescale）、`boolean shade`。
  - `Deserializer` 解析 `from`/`to`/`rotation`/`shade`/`faces`。

- **`BlockElementFace`**（描述单个面）
  - 字段：`Direction cullForDirection`（该面朝向被遮挡时剔除，JSON 中 `cullface`）、`int tintIndex`（染色索引，默认 `-1` = NO_TINT）、`String texture`（纹理引用键）、`BlockFaceUV uv`。
  - `Deserializer` 解析 `cullface`/`tintindex`/`texture`，并通过 `context.deserialize(jsonobject, BlockFaceUV.class)` 解析 UV。

- **`BlockFaceUV`**（UV 坐标 + 旋转）
  - 字段：`float[] uvs`（4 个值 u0/v0/u1/v1，单位像素）、`int rotation`（仅允许 0/90/180/270）。
  - `getU(int index)`/`getV(int index)` 按 `rotation` 偏移取值；`getReverseIndex` 反向偏移。
  - `Deserializer` 解析 `uv` 数组（必须 4 个值）和 `rotation`。

> **数据流**：`assets/.../models/block/xxx.json` → `GSON.fromJson` → `BlockModel{ elements:[ BlockElement{ from, to, faces:{ Direction: BlockElementFace{ cullForDirection, tintIndex, texture, uv: BlockFaceUV{ uvs, rotation } } }, rotation, shade } ], textureMap, parent, ... }`

### 1.2 模型烘焙

烘焙把"逻辑模型"转成可直接绘制的 `BakedQuad` 集合（`BakedModel`）。

- **`BlockModel.bakeVanilla(...)`**（Forge 中 `bake` 委托给 `UnbakedGeometryHelper.bake`，内部最终调用 `bakeVanilla`）：
  - 取 particle 纹理；若根模型是 `BLOCK_ENTITY_MARKER` 则返回 `BuiltInModel`（由 BlockEntityRenderer 渲染）。
  - 否则构造 `SimpleBakedModel.Builder`，遍历 `getElements()` 中每个 `BlockElement`，再遍历其 `faces`：
    ```java
    for (BlockElement blockelement : this.getElements()) {
        for (Direction direction : blockelement.faces.keySet()) {
            BlockElementFace face = blockelement.faces.get(direction);
            TextureAtlasSprite sprite = spriteGetter.apply(this.getMaterial(face.texture));
            if (face.cullForDirection == null)
                builder.addUnculledFace(bakeFace(blockelement, face, sprite, direction, state, location));
            else
                builder.addCulledFace(Direction.rotate(state.getRotation().getMatrix(), face.cullForDirection),
                                      bakeFace(blockelement, face, sprite, direction, state, location));
        }
    }
    ```
  - `cullForDirection != null` 的面按朝向放入 `culledFaces[Direction]`（用于面剔除），其余放入 `unculledFaces`。
  - `BlockModel.bakeFace(...)` 静态方法委托给 `FaceBakery.bakeQuad(...)`。

- **`FaceBakery`**（`net/minecraft/client/renderer/block/model/FaceBakery.java`）—— 真正生成顶点数据：
  - 常量：`VERTEX_INT_SIZE = 8`（每顶点 8 个 int = 32 字节）、`VERTEX_COUNT = 4`。
  - **`bakeQuad(posFrom, posTo, face, sprite, facing, transform, partRotation, shade, modelLocation)`**：
    1. 处理 UV Lock（`transform.isUvLocked()` → `recomputeUVs`）。
    2. UV 收缩防溢出（`sprite.uvShrinkRatio()` + `lerp`）。
    3. 调 `makeVertices(...)` 生成 `int[32]`。
    4. `calculateFacing(aint)` 根据顶点反推朝向。
    5. `ForgeHooksClient.fillNormal(aint, direction)` 填充法线 `[未确认 fillNormal 内部，由 Forge 实现，写入每顶点第 7 个 int]`。
    6. 构造 `new BakedQuad(aint, face.tintIndex, direction, sprite, shade, data.ambientOcclusion())`。
  - **`makeVertices(...)`**：`int[32]`，循环 4 个顶点调 `bakeVertex`。
  - **`bakeVertex(...)`**：用 `FaceInfo.VertexInfo` 取该朝向该顶点索引对应的 xyz 坐标分量；`applyElementRotation`（元素局部旋转，绕 `partRotation.origin`）→ `applyModelRotation`（模型整体变换，绕中心 `(0.5,0.5,0.5)`）→ `fillVertex`。
  - **`fillVertex(vertexData, vertexIndex, vector, sprite, blockFaceUV)`**：
    ```java
    int i = vertexIndex * 8;
    vertexData[i]   = Float.floatToRawIntBits(vector.x());  // position x
    vertexData[i+1] = Float.floatToRawIntBits(vector.y());  // position y
    vertexData[i+2] = Float.floatToRawIntBits(vector.z());  // position z
    vertexData[i+3] = -1;                                   // color = 0xFFFFFFFF (白)
    vertexData[i+4] = Float.floatToRawIntBits(sprite.getU(...));  // uv0 u
    vertexData[i+5] = Float.floatToRawIntBits(sprite.getV(...));  // uv0 v
    // i+6 (normal+padding) 由 fillNormal 写入；i+6 的 uv2(光图)在烘焙时不写，绘制时填
    ```
  - `setupShape` 把 from/to 除以 16 归一化；`applyElementRotation`/`rotateVertexBy` 用 `Matrix4f` 做仿射变换。

- **`BakedQuad`**（`net/minecraft/client/renderer/block/model/BakedQuad.java`）：
  - 字段：`int[] vertices`（32 个 int，即 4 顶点 × 8 int）、`int tintIndex`、`Direction direction`、`TextureAtlasSprite sprite`、`boolean shade`、`boolean hasAmbientOcclusion`。
  - 注释明确："Joined 4 vertex records... Vanilla minecraft uses DefaultVertexFormats.BLOCK"。

- **`BlockModelShaper`**（`net/minecraft/client/renderer/block/BlockModelShaper.java`）：
  - 维护 `Map<BlockState, BakedModel> modelByStateCache`（由 ModelManager 在 reload 时通过 `replaceCache` 填充）。
  - `getBlockModel(state)` 从缓存取 BakedModel，缺失返回 missing model。
  - `stateToModelLocation` 把 BlockState 转成 `ModelResourceLocation`（如 `minecraft:stone#`）。

- **`BlockRenderDispatcher`**（`net/minecraft/client/renderer/block/BlockRenderDispatcher.java`）：
  - 持有 `BlockModelShaper`、`ModelBlockRenderer`、`LiquidBlockRenderer`、`BlockEntityWithoutLevelRenderer`、`BlockColors`。
  - `renderBatched(state, pos, level, poseStack, consumer, checkSides, random, modelData, renderType)` → 内部调用 `ModelBlockRenderer.tesselateBlock`。
  - `renderLiquid(...)` → `LiquidBlockRenderer.tesselate`。
  - `renderSingleBlock(...)` 用于物品/GUI 中渲染单个方块状态。

#### 1.2.1 ModelBlockRenderer 如何把 BakedQuad 写入顶点缓冲

- **`ModelBlockRenderer.tesselateBlock(level, model, state, pos, poseStack, consumer, checkSides, random, seed, packedOverlay, modelData, renderType)`**：
  - 判断是否启用环境光（`useAmbientOcclusion` 且方块不发光）：是→`tesselateWithAO`，否→`tesselateWithoutAO`。
  - 应用方块偏移 `state.getOffset`（如沙子下滑）。
- **`tesselateWithAO` / `tesselateWithoutAO`**：对 6 个方向 + null 方向，调 `model.getQuads(state, direction, random, modelData, renderType)` 取四边形列表，做面剔除检查后调 `renderModelFaceAO` / `renderModelFaceFlat`。
- **`renderModelFaceFlat(...)`**：对每个 quad，`level.getShade(direction, shade)` 计算方向着色，调 `putQuadData`。
- **`renderModelFaceAO(...)`**：计算形状/相邻方块，`AmbientOcclusionFace.calculate` 算出每顶点亮度 `brightness[4]` 与光图 `lightmap[4]`，再 `putQuadData`。
- **`putQuadData(...)`**：
  ```java
  if (quad.isTinted()) { /* 取 BlockColors 染色 → r,g,b */ } else { r=g=b=1; }
  consumer.putBulkData(pose, quad, new float[]{b0,b1,b2,b3}, r,g,b,
                       new int[]{lm0,lm1,lm2,lm3}, packedOverlay, true);
  ```
- **`renderModel(pose, consumer, state, model, r,g,b, packedLight, packedOverlay, ...)`**：无世界光照信息的简化版本（用于物品/BlockEntity），直接 `renderQuadList` → `consumer.putBulkData(pose, quad, r,g,b, packedLight, packedOverlay)`。

> 关键：`BakedQuad` 的 `int[32]` 是**已经过烘焙变换**的模型空间顶点；运行时只做 `PoseStack` 变换、染色、填光图、填 overlay，再写入。

### 1.3 区块网格化层

区块重建把一个 16×16×16 区域内所有方块的几何组装进每个 `RenderType` 对应的 `VertexBuffer`。

- **`ChunkRenderDispatcher`**（`net/minecraft/client/renderer/chunk/ChunkRenderDispatcher.java`）：
  - 持有 `Queue<ChunkBufferBuilderPack> freeBuffers`（线程池可用的缓冲包，数量按内存/核数计算）、`fixedBuffers`、`ClientLevel level`、`LevelRenderer renderer`。
  - `uploadChunkLayer(BufferBuilder.RenderedBuffer, VertexBuffer)`：在渲染线程把构建好的 buffer 上传到指定 VertexBuffer。

- **`ChunkRenderDispatcher.RenderChunk`**（内部类，真正的渲染区块，注意与同包的 `RenderChunk.java` 包装类区分）：
  - **`Map<RenderType, VertexBuffer> buffers`**：对 `RenderType.chunkBufferLayers()` 每个 RenderType 创建一个 `new VertexBuffer(Usage.STATIC)`。
  - `AtomicReference<CompiledChunk> compiled`：当前编译结果。
  - `beginLayer(BufferBuilder builder)`：`builder.begin(VertexFormat.Mode.QUADS, DefaultVertexFormat.BLOCK)` —— 所有方块层都用 QUADS + BLOCK 格式。
  - `getBuffer(RenderType)` 取对应 VertexBuffer；`resortTransparency` 触发半透明重排序。

- **`ChunkRenderDispatcher.CompiledChunk`**：`Set<RenderType> hasBlocks`、`List<BlockEntity> renderableBlockEntities`、`VisibilitySet visibilitySet`、`BufferBuilder.SortState transparencyState`。

- **`ChunkRenderDispatcher.RenderChunk.RebuildTask.compile(x, y, z, chunkBufferBuilderPack)`**（核心网格化逻辑）：
  ```
  ModelBlockRenderer.enableCaching();
  Set<RenderType> set = ...;
  for (BlockPos in [origin, origin+15]) {
      BlockState blockstate = region.getBlockState(pos);
      // 1. 流体
      FluidState fluidstate = blockstate.getFluidState();
      if (!fluidstate.isEmpty()) {
          RenderType rendertype = ItemBlockRenderTypes.getRenderLayer(fluidstate);   // 通常 translucent
          BufferBuilder bb = chunkBufferBuilderPack.builder(rendertype);
          if (set.add(rendertype)) RenderChunk.this.beginLayer(bb);   // 首次使用该层才 begin
          blockrenderdispatcher.renderLiquid(pos, region, bb, blockstate, fluidstate);
      }
      // 2. 方块
      if (blockstate.getRenderShape() != INVISIBLE) {
          BakedModel model = blockrenderdispatcher.getBlockModel(blockstate);
          for (RenderType rt : model.getRenderTypes(blockstate, random, modelData)) {
              BufferBuilder bb = chunkBufferBuilderPack.builder(rt);
              if (set.add(rt)) RenderChunk.this.beginLayer(bb);
              poseStack.pushPose();
              poseStack.translate(pos&15, pos&15, pos&15);   // 区块局部坐标
              blockrenderdispatcher.renderBatched(blockstate, pos, region, poseStack, bb, true, random, modelData, rt);
              poseStack.popPose();
          }
      }
  }
  // 3. 半透明按距离排序
  if (set.contains(translucent)) {
      bb1.setQuadSorting(VertexSorting.byDistance(...));   // 透明四边形从远到近排序
      transparencyState = bb1.getSortState();
  }
  // 4. 结束每层
  for (RenderType rt : set) {
      RenderedBuffer r = builder(rt).endOrDiscardIfEmpty();   // 空层丢弃
      if (r != null) results.renderedLayers.put(rt, r);
  }
  ```
  结果（`renderedLayers`、`blockEntities`）在 `doTask` 完成后通过 `uploadChunkLayer` 上传到 `RenderChunk.buffers`，并更新 `CompiledChunk`。

> **数据流**：BlockState → `BakedModel.getRenderTypes` 决定写哪个 `BufferBuilder`（即哪个 RenderType 层）→ `ModelBlockRenderer.tesselateBlock` 写入 → `endOrDiscardIfEmpty` 成 `RenderedBuffer` → `VertexBuffer.upload`。

### 1.4 流体渲染

- **`LiquidBlockRenderer`**（`net/minecraft/client/renderer/block/LiquidBlockRenderer.java`）：
  - 字段：`TextureAtlasSprite[] lavaIcons[2]`（静止/流动）、`waterIcons[2]`、`waterOverlay`。
  - `setupSprites()` 在 reload 时从 `ModelBakery.LAVA_FLOW`/`WATER_FLOW`/`WATER_OVERLAY` 取精灵。
  - **`tesselate(level, pos, vertexConsumer, blockState, fluidState)`**：
    - 取流体颜色与 alpha（`IClientFluidTypeExtensions.getTintColor`）。
    - 计算四个角的高度（`getHeight`/`calculateAverageHeight`，相邻流体插值，`MAX_FLUID_HEIGHT = 0.8888889F`）。
    - 对每个可见面（上/下/北/南/东/西，`shouldRenderFace` 判断）调用 `vertex(...)` 直接写顶点到 `vertexConsumer`（即 `BufferBuilder`），不经过 `BakedQuad`。
  - 流体的 RenderType 由 `ItemBlockRenderTypes.getRenderLayer(fluidState)` 决定（水/岩浆均为 `translucent`）。

### 1.5 物品中方块的渲染类型

- **`ItemBlockRenderTypes`**（`net/minecraft/client/renderer/ItemBlockRenderTypes.java`）：
  - `Map<Block, RenderType> TYPE_BY_BLOCK`：硬编码映射（草方块/树叶→`cutoutMipped`，玻璃/花草/栅栏门→`cutout`，冰/下界传送门/染色玻璃→`translucent`，绊线→`tripwire`，其余默认 `solid`）。
  - `Map<Fluid, RenderType> TYPE_BY_FLUID`：水（流动/静止）→ `translucent`。
  - `getChunkRenderType(BlockState)` / `getRenderLayers(BlockState)`：返回区块层 RenderType（`solid`/`cutout`/`cutoutMipped`/`translucent`/`tripwire`）。
  - `getRenderLayer(FluidState)`：流体用。
  - `getRenderType(BlockState, boolean cull)` / `getRenderType(ItemStack, boolean cull)`：返回实体/物品形态的 RenderType（映射到 `Sheets` 的 block sheet）。
  - 树叶：`renderCutout` 为真用 `cutoutMipped`，否则 `solid`（`setFancy` 控制）。
  - `[Forge 改动]` 1.19+ 起推荐在模型 JSON 写 `"render_type"` 或重写 `BakedModel.getRenderTypes`，本类的硬编码表已 `@Deprecated`。

## 2. 实体几何管线

### 2.1 模型定义层

实体模型是**程序化定义**的（非 JSON），通过 builder API 构建层级树，注册后在资源重载时烘焙为 `ModelPart`。

#### 2.1.1 定义与注册

- **`LayerDefinitions`**（`net/minecraft/client/model/geom/LayerDefinitions.java`）：
  - 静态 `createRoots()` 返回 `Map<ModelLayerLocation, LayerDefinition>`，内含所有原版实体模型的定义（用 `MeshDefinition` + `CubeListBuilder` + `PartPose` 构建）。
- **`EntityModelSet`**（`net/minecraft/client/model/geom/EntityModelSet.java`）：
  - `onResourceManagerReload` 时调用 `LayerDefinitions.createRoots()` 存入 `roots`。
  - `bakeLayer(ModelLayerLocation)` → `LayerDefinition.bakeRoot()` 返回 `ModelPart`。

#### 2.1.2 Builder 体系

- **`MeshDefinition`**：含一个根 `PartDefinition`（`PartPose.ZERO`）。
- **`PartDefinition`**（`net/minecraft/client/model/geom/builders/PartDefinition.java`）：
  - `List<CubeDefinition> cubes`、`PartPose partPose`、`Map<String, PartDefinition> children`。
  - `addOrReplaceChild(name, cubes, partPose)` 添加子部件。
  - **`bake(texWidth, texHeight)`**：递归烘焙子部件 → 烘焙 cubes → 构造 `new ModelPart(cubes, children)` → `setInitialPose` + `loadPose`。
- **`CubeDefinition`** → `bake` 产生 `ModelPart.Cube`。
- **`LayerDefinition`**：持有 `MeshDefinition` + `MaterialDefinition`（纹理宽高）；`bakeRoot()` → `mesh.getRoot().bake(texW, texH)`。
- **`PartPose`**（`net/minecraft/client/model/geom/PartPose.java`）：不可变值对象 `{ x, y, z, xRot, yRot, zRot }`，工厂 `offset`/`rotation`/`offsetAndRotation`。

#### 2.1.3 ModelPart 运行时结构

- **`ModelPart`**（`net/minecraft/client/model/geom/ModelPart.java`，`final`）：
  - 字段：`x,y,z`（平移，1/16 块）、`xRot,yRot,zRot`（旋转弧度）、`xScale,yScale,zScale`（默认 1）、`boolean visible`、`boolean skipDraw`、`List<Cube> cubes`、`Map<String,ModelPart> children`、`PartPose initialPose`。
  - **`translateAndRotate(PoseStack)`**：`translate(x/16, y/16, z/16)` → 若有旋转 `mulPose(Quaternionf.rotationZYX(zRot,yRot,xRot))` → 若有缩放 `scale(...)`。
  - **`render(poseStack, vertexConsumer, packedLight, packedOverlay, r,g,b,a)`**：
    ```
    if (visible) {
        poseStack.pushPose();
        translateAndRotate(poseStack);
        if (!skipDraw) compile(poseStack.last(), vertexConsumer, ...);
        for (child : children.values()) child.render(...);   // 递归
        poseStack.popPose();
    }
    ```
  - **`compile(...)`**：对每个 `Cube` 调 `cube.compile(...)`。
- **`ModelPart.Cube`**：
  - 构造时根据尺寸/grow/mirror/可见面**预计算** `Polygon[] polygons`（每个面一个 4 顶点多边形，UV 已按纹理宽高归一化）。
  - **`compile(PoseStack.Pose pose, VertexConsumer vc, packedLight, packedOverlay, r,g,b,a)`**：
    ```java
    Matrix4f matrix4f = pose.pose();
    Matrix3f matrix3f = pose.normal();
    for (Polygon polygon : polygons) {
        Vector3f n = matrix3f.transform(polygon.normal);   // 法线变换
        for (Vertex v : polygon.vertices) {
            Vector4f pos = matrix4f.transform(new Vector4f(v.pos.x/16, v.pos.y/16, v.pos.z/16, 1));
            vertexConsumer.vertex(pos.x, pos.y, pos.z, r,g,b,a, v.u, v.v, packedOverlay, packedLight, n.x,n.y,n.z);
        }
    }
    ```
    ——每个顶点直接调用 14 参数的 `vertex(...)`（对应 `NEW_ENTITY` 格式：position+color+uv0+uv1(overlay)+uv2(lightmap)+normal）。
  - `ModelPart.Polygon`：`Vertex[4]` + `Vector3f normal`（构造时按方向算 UV，mirror 时翻转顶点顺序与法线 x）。
  - `ModelPart.Vertex`：`Vector3f pos` + `float u,v`。

#### 2.1.4 EntityModel 抽象

- **`Model`**（基类）：`Function<ResourceLocation, RenderType> renderType`；抽象 `renderToBuffer(poseStack, buffer, packedLight, packedOverlay, r,g,b,a)`；`renderType(location)` 应用函数得到 RenderType。
- **`EntityModel<T extends Entity>`**：字段 `attackTime`/`riding`/`young`；默认 renderType 函数为 `RenderType::entityCutoutNoCull`；抽象 `setupAnim(entity, limbSwing, limbSwingAmount, ageInTicks, netHeadYaw, headPitch)` 摆姿势；`prepareMobModel` 钩子。
- **`HierarchicalModel<E>`**（现代基类）：`renderToBuffer` → `root().render(poseStack, buffer, ...)`（即委托给 ModelPart 树）；抽象 `root()`；提供 `animate`/`animateWalk`（基于 `AnimationDefinition` 关键帧动画）。

### 2.2 渲染流程

- **`EntityRenderer<T>`**（基类，`net/minecraft/client/renderer/entity/EntityRenderer.java`）：持有 `EntityRenderDispatcher`、`shadowRadius`；抽象 `getTextureLocation(T)`；`render(...)` 基类版本绘制名字标签/阴影。
- **`LivingEntityRenderer<T extends LivingEntity, M extends EntityModel<T>>`**（`net/minecraft/client/renderer/entity/LivingEntityRenderer.java`）：
  - 字段：`M model`、`List<RenderLayer<T,M>> layers`（叠加层）。
  - **`render(entity, entityYaw, partialTicks, poseStack, buffer, packedLight)`** 流程：
    1. `poseStack.pushPose()`。
    2. 设置模型状态：`model.attackTime`/`riding`/`young`；计算 `yBodyRot`/`yHeadRot` 插值、`xRot` 插值。
    3. 睡眠方向平移；`getBob`（受伤晃动）；`setupRotations(entity, poseStack, bob, bodyYaw, partialTicks)`。
    4. **`poseStack.scale(-1, -1, 1)`**（实体模型坐标系 y 向上，需翻转）；`scale(entity)`（如小僵尸缩放）；`translate(0, -1.501, 0)`（模型原点在脚底，下移 1.5）。
    5. 计算 `limbSwing`/`limbSwingAmount`；`model.prepareMobModel` + **`model.setupAnim(...)`**（摆姿势）。
    6. 确定 RenderType：`getRenderType(entity, bodyVisible, translucent, glowing)` →
       - 不可见半透明：`RenderType.itemEntityTranslucentCull(texture)`；
       - 可见：`model.renderType(texture)`（默认 `entityCutoutNoCull`）；
       - 发光轮廓：`RenderType.outline(texture)`。
    7. `VertexConsumer vc = buffer.getBuffer(renderType)`；
       `int overlay = getOverlayCoords(entity, whiteOverlayProgress)`（受伤红闪/死亡用 overlay）。
    8. **`model.renderToBuffer(poseStack, vc, packedLight, overlay, 1,1,1, invisible?0.15:1)`** —— 写入所有顶点。
    9. 非旁观者：遍历 `layers`，每个 `RenderLayer.render(...)` 叠加。
    10. `poseStack.popPose()`；`super.render(...)`（名字/阴影）。
- **`EntityRenderDispatcher`**（`net/minecraft/client/renderer/entity/EntityRenderDispatcher.java`）：
  - `getRenderer(entity)` 按 Entity 类型查表（玩家走 `playerRenderers`）。
  - **`render(entity, x,y,z, rotationYaw, partialTicks, poseStack, buffer, packedLight)`**：平移到实体插值位置 → `entityrenderer.render(...)` → 火焰动画 → 平移回。
  - `renderShadow(...)` 绘制阴影（用 `RenderType.entityShadow(shadow.png)`）。
  - 维护 `Map<EntityType, EntityRenderer>` / `playerRenderers`，由 `EntityRenderers.register` 在注册期填充。

### 2.3 渲染层叠加

- **`RenderLayer<T extends LivingEntity, M extends EntityModel<T>>`**（`net/minecraft/client/renderer/entity/layers/RenderLayer.java`，抽象）：
  - 持有 `RenderLayerParent`（提供 `getModel`/`getTextureLocation`）。

- 常见叠加层（`net/minecraft/client/renderer/entity/layers/`）：

  - `ElytraLayer`：鞘翅。
  - `CapeLayer`：披风。

  - `ItemInHandLayer`/`HeldItemLayer`：手持物品。
  - `EyesLayer`/`EnderEyesLayer`：发光眼睛（用 emissive RenderType）。
  - `ArrowLayer`/`BeeStingerLayer`：身上插的箭。
  - `SaddleLayer`/`HorseArmorLayer`/`SheepFurLayer`：覆盖层模型。
- 叠加层在 `LivingEntityRenderer.render` 主模型之后依次调用，可使用自己的 RenderType（通过 `buffer.getBuffer(...)` 获取新的 `VertexConsumer`）。

### 2.4 BlockEntity 渲染（简要）

- **`BlockEntityRenderer`**（`net/minecraft/client/renderer/blockentity/BlockEntityRenderer.java`，接口）：`render(blockEntity, partialTick, poseStack, bufferSource, packedLight, packedOverlay)`。

## 3. 共享渲染基础设施

### 3.1 RenderType 系统

- **`RenderType`**（`net/minecraft/client/renderer/RenderType.java`，`extends RenderStateShard`）：
  - 核心字段：`VertexFormat format`、`VertexFormat.Mode mode`、`int bufferSize`、`boolean affectsCrumbling`、`boolean sortOnUpload`、setup/clear state 的 `Runnable`。
  - 构造：`RenderType(name, format, mode, bufferSize, affectsCrumbling, sortOnUpload, setupState, clearState)`。
  - **`create(name, format, mode, bufferSize, affectsCrumbling, sortOnUpload, CompositeState)`** → `new CompositeRenderType(...)`。
  - **`end(BufferBuilder, VertexSorting)`**（关键，触发绘制）：
    ```java
    if (building) {
        if (sortOnUpload) bufferBuilder.setQuadSorting(quadSorting);
        RenderedBuffer r = bufferBuilder.end();
        this.setupRenderState();               // 设置 GL 状态（含 shader/texture/blend）
        BufferUploader.drawWithShader(r);      // 上传 + 用当前 shader 绘制
        this.clearRenderState();
    }
    ```
- **`CompositeRenderType`**：setup/clear 委托给 `CompositeState.states.forEach(RenderStateShard::setupRenderState/clearRenderState)`。
- **`CompositeState`**：13 个 `RenderStateShard` 子状态（构造时聚合成 `ImmutableList<RenderStateShard> states`）：
  - `textureState`（`TextureStateShard`）、`shaderState`（`ShaderStateShard`）、`transparencyState`、`depthTestState`、`cullState`、`lightmapState`、`overlayState`、`layeringState`、`outputState`、`texturingState`、`writeMaskState`、`colorLogicState`、`lineState`。
  - `CompositeStateBuilder` 提供默认值（如 `NO_TRANSPARENCY`、`LEQUAL_DEPTH_TEST`、`CULL`、`COLOR_DEPTH_WRITE`）。
- **区块层 RenderType**（`CHUNK_BUFFER_LAYERS = ImmutableList.of(solid(), cutoutMipped(), cutout(), translucent(), tripwire())`）：
  - `solid`：BLOCK 格式，`BLOCK_SHEET_MIPPED` 纹理，`RENDERTYPE_SOLID_SHADER`，`sortOnUpload=false`。
  - `cutoutMipped`：BLOCK，`BLOCK_SHEET_MIPPED`，`RENDERTYPE_CUTOUT_MIPPED_SHADER`。
  - `cutout`：BLOCK，`BLOCK_SHEET`（无 mipmap），`RENDERTYPE_CUTOUT_SHADER`。
  - `translucent`：BLOCK，`translucentState(RENDERTYPE_TRANSLUCENT_SHADER)`，`sortOnUpload=true`（需排序）。
- **实体常用 RenderType**（均 `NEW_ENTITY` 格式 + QUADS + 缓冲 256，按纹理 `ResourceLocation` memoize）：
  - `entitySolid(texture)`：`NO_TRANSPARENCY` + `LIGHTMAP` + `OVERLAY`，`RENDERTYPE_ENTITY_SOLID_SHADER`。
  - `entityCutout(texture)`：`RENDERTYPE_ENTITY_CUTOUT_SHADER`。
  - `entityCutoutNoCull(texture, outline)`：`NO_CULL` + `OVERLAY`，`RENDERTYPE_ENTITY_CUTOUT_NO_CULL_SHADER`。
  - `entityTranslucent(texture, outline)`：`TRANSLUCENT_TRANSPARENCY` + `NO_CULL`，`RENDERTYPE_ENTITY_TRANSLUCENT_SHADER`。
  - `entityTranslucentCull`、`itemEntityTranslucentCull`、`entityTranslucentEmissive`、`armorCutoutNoCull` 等。
- **`RenderStateShard`**（`net/minecraft/client/renderer/RenderStateShard.java`）：
  - 每个 shard 持有 `setupState`/`clearState` 两个 `Runnable`，`setupRenderState()`/`clearRenderState()` 调用之。
  - 预定义静态实例：
    - `TRANSLUCENT_TRANSPARENCY`：`enableBlend` + `blendFuncSeparate(SRC_ALPHA, ONE_MINUS_SRC_ALPHA, ONE, ONE_MINUS_SRC_ALPHA)`。
    - `NO_TRANSPARENCY`：`disableBlend`。
    - `LIGHTMAP`/`NO_LIGHTMAP`、`OVERLAY`/`NO_OVERLAY`、`CULL`/`NO_CULL`、`LEQUAL_DEPTH_TEST` 等。
    - `BLOCK_SHEET_MIPPED`/`BLOCK_SHEET`：`TextureStateShard(TextureAtlas.LOCATION_BLOCKS, blur, mipmap)`。
  - **`ShaderStateShard`**：setup 调 `RenderSystem.setShader(supplier)`（仅设置"当前 shader 引用"，真正 apply 在绘制时）。
  - **`TextureStateShard`**：setup 调 `textureManager.getTexture(rl).setFilter(blur, mipmap)` + `RenderSystem.setShaderTexture(0, rl)`。

### 3.2 顶点缓冲系统

（`com/mojang/blaze3d/vertex/`）

- **`VertexFormatElement`**：`Type`（FLOAT/UBYTE/BYTE/SHORT/...）+ `Usage`（POSITION/COLOR/UV/NORMAL/PADDING）+ `index`（UV 有 0/1/2 三个通道）+ `count`。
  - `Usage.setupBufferState` 在 VAO 启用时配置对应的 `glVertexAttribPointer`。
- **`DefaultVertexFormat`**（预定义顶点布局，`VertexFormatElement` 组合）：
  - **`BLOCK`**（方块用，32 字节 = 8 int）：`Position(3 FLOAT)` + `Color(4 UBYTE)` + `UV0(2 FLOAT)` + `UV2(2 SHORT 光图)` + `Normal(3 BYTE)` + `Padding(1 BYTE)`。
  - **`NEW_ENTITY`**（实体用，36 字节）：`Position(3F)` + `Color(4UB)` + `UV0(2F)` + `UV1(2 SHORT overlay)` + `UV2(2 SHORT 光图)` + `Normal(3B)` + `Padding(1B)`。
  - 其他：`POSITION`、`POSITION_COLOR`、`POSITION_COLOR_NORMAL`、`POSITION_TEX_COLOR_NORMAL`、`PARTICLE`、`BLIT_SCREEN` 等。
- **`VertexFormat`**：`ImmutableList<VertexFormatElement> elements`、`vertexSize`、`offsets`；`getIntegerSize = vertexSize/4`；`setupBufferState`/`clearBufferState` 配置 VAO 属性；持有 `immediateDrawVertexBuffer`（即时绘制复用的 VBO）。
- **`VertexConsumer`**（接口）：链式 API `vertex/color/uv/overlayCoords/uv2/normal/endVertex`；`vertex(Matrix4f,...)`/`normal(Matrix3f,...)`；`putBulkData(...)`（批量写入 BakedQuad）。
  - **`putBulkData(pose, quad, colorMuls[4], r,g,b, alpha, lightmaps[4], overlay, mulColor)`**（核心批量写入）：
    - 取 `quad.getVertices()`（int[32]），每顶点 8 int；
    - 用 `MemoryStack` 临时 `ByteBuffer`（BLOCK vertex size）；
    - 对 4 顶点：读 position(3 float) → 经 `pose.pose()` 变换；读 color → 按 `mulColor` 乘；`applyBakedLighting` 应用烘焙光图；读 uv0 → 经 sprite；`applyBakedNormals` 应用烘焙法线；调 `this.vertex(x,y,z, r,g,b,a, u,v, overlay, lightmap, nx,ny,nz)`（单次写满 NEW_ENTITY 的全部字段）。
- **`BufferVertexConsumer`**（接口，extends VertexConsumer）：基于 `currentElement()`/`nextElement()`/`putByte`/`putShort`/`putFloat` 的元素顺序写入实现 `vertex/color/uv/uv2/normal` 等。
- **`DefaultedVertexConsumer`**（抽象）：默认颜色支持。
- **`BufferBuilder`**（`extends DefaultedVertexConsumer implements BufferVertexConsumer`）：
  - `ByteBuffer buffer`（`MemoryTracker.create` 分配的堆外内存，按 2MB 增长）。
  - `begin(Mode, VertexFormat)`：设置 building/format/mode，`currentElement` 指向第一个元素。
  - `putByte/putShort/putFloat(index, ...)`：直接写 `buffer` 的 `nextElementByte + index`。
  - `endVertex()`：`vertices++`，校验所有元素已填，必要时扩容（LINES 模式复制顶点）。
  - `nextElement()`：推进到下一个元素，跳过 PADDING，应用 defaultColor。
  - **`vertex(float x,y,z, r,g,b,a, u,v, overlay, lightmap, nx,ny,nz)`**（14 参数，fastFormat 时一次性写入，避免逐元素调用）。
  - **`end()` / `endOrDiscardIfEmpty()`**：调 `storeRenderedBuffer` → 生成 `RenderedBuffer(startOffset, DrawState)`。
  - **`storeRenderedBuffer()`**：算 indexCount；若 `sortingPoints != null` 调 `putSortedQuadIndices`（按距离排序四边形索引）；构造 `DrawState(format, vertices, indexCount, mode, indexType, indexOnly, sequentialIndex)`。
  - `setQuadSorting(VertexSorting)` / `getSortState()` / `restoreSortState()`：半透明排序支持。
- **`BufferUploader`**（即时绘制）：
  - `drawWithShader(RenderedBuffer)` → `_drawWithShader`：`upload(buffer)`（按 format 取/绑定 `immediateDrawVertexBuffer`，`VertexBuffer.upload`）→ `vertexbuffer.drawWithShader(modelView, projection, shader)`。
  - `draw(RenderedBuffer)`：不带 shader 的绘制。
  - `lastImmediateBuffer` 跟踪当前绑定的即时 VBO 以减少切换。
- **`VertexBuffer`**（GPU VBO，`com/mojang/blaze3d/vertex/VertexBuffer.java`）：
  - GL 对象：`vertexBufferId`（GL_ARRAY_BUFFER）、`indexBufferId`（GL_ELEMENT_ARRAY_BUFFER）、`arrayObjectId`（VAO）。
  - **`upload(RenderedBuffer)`**：`uploadVertexBuffer(drawState, vertexBuffer)` + `uploadIndexBuffer(drawState, indexBuffer)`，记录 `indexCount`/`indexType`/`mode`。
  - **`uploadVertexBuffer`**：`_glBindBuffer(GL_ARRAY_BUFFER, id)` + 若 format 变更则 `format.setupBufferState()`；`RenderSystem.glBufferData(GL_ARRAY_BUFFER, buffer, usage)`。
  - **`uploadIndexBuffer`**：若非顺序索引则上传到 `GL_ELEMENT_ARRAY_BUFFER`；若顺序索引则用 `RenderSystem.getSequentialBuffer(mode)`（共享的索引缓冲）。
  - **`bind()`**：`_glBindVertexArray(arrayObjectId)`（绑定 VAO，属性指针已预配置）。
  - **`draw()`**：`RenderSystem.drawElements(mode.asGLMode, indexCount, indexType.asGLType)`（即 `glDrawElements`）。
  - **`drawWithShader(modelView, projection, shader)`** → **`_drawWithShader`**：设置 12 个 sampler、`MODEL_VIEW_MATRIX`、`PROJECTION_MATRIX`、`INVERSE_VIEW_ROTATION_MATRIX`、`COLOR_MODULATOR`、`GLINT_ALPHA`、`FOOG_START/END/COLOR/SHAPE`、`TEXTURE_MATRIX`、`GAME_TIME`、`SCREEN_SIZE`、`LINE_WIDTH` 等 Uniform，`setupShaderLights`，`shader.apply()` → `draw()` → `shader.clear()`。
- **`MultiBufferSource`**（接口）/ **`BufferSource`**（实现，实体/物品绘制用）：
  - `BufferBuilder builder`（共享后备缓冲）+ `Map<RenderType, BufferBuilder> fixedBuffers`（固定层缓冲）。
  - **`getBuffer(RenderType)`**：取对应 `BufferBuilder`；若与上一 RenderType 不同（或不可合并）则 flush 上一批；首次使用调 `builder.begin(mode, format)`；返回 `VertexConsumer`。
  - **`endBatch(RenderType)`**：`renderType.end(bufferbuilder, vertexSorting)`（触发 `BufferUploader.drawWithShader`）。
  - **`endBatch()`**：flush 共享缓冲 + 所有 fixedBuffers。
  - `OutlineBufferSource`：包裹一个 BufferSource，同时写一份到轮廓 RenderType（用于发光实体）。

### 3.3 着色器系统

- **`ShaderInstance`**（`net/minecraft/client/renderer/ShaderInstance.java`，`implements Shader`）：
  - 构造 `ShaderInstance(ResourceProvider, ResourceLocation, VertexFormat)`：
    - 读 `shaders/core/<name>.json`，解析 `vertex`/`fragment` 程序名、`samplers`、`attributes`、`blend`（`BlendMode`）。
    - 用 `Program` 编译 vertex/fragment GLSL（支持 `#include`，经 `GlslPreprocessor`），链接成 `programId`。
    - `updateLocations()`：查询所有 sampler/uniform 的 location，构造 `Uniform` 对象存入 `uniformMap`。
  - **预定义 Uniform 字段**（按名查找，shader 中不存在则为 null）：`MODEL_VIEW_MATRIX`、`PROJECTION_MATRIX`、`INVERSE_VIEW_ROTATION_MATRIX`、`TEXTURE_MATRIX`、`SCREEN_SIZE`、`COLOR_MODULATOR`、`LIGHT0_DIRECTION`、`LIGHT1_DIRECTION`、`GLINT_ALPHA`、`FOG_START/END/COLOR/SHAPE`、`LINE_WIDTH`、`GAME_TIME`、`CHUNK_OFFSET`（区块专用，每区块绘制前设偏移）。
  - **`apply()`**：`blend.apply()` → 若 programId 变化则 `ProgramManager.glUseProgram(programId)` → 绑定所有 sampler 纹理 → 上传所有 Uniform。
  - **`clear()`**：`glUseProgram(0)`，解绑 sampler 纹理。
  - `getUniform(name)` / `safeGetUniform(name)`（不存在返回 `DUMMY_UNIFORM`）。
- **着色器与 RenderType 的关系**：`RenderType` 通过 `ShaderStateShard` 持有 `Supplier<ShaderInstance>`（如 `RENDERTYPE_SOLID_SHADER = new ShaderStateShard(GameRenderer::getRendertypeSolidShader)`）。`setupRenderState` 时 `RenderSystem.setShader(supplier)` 设置当前 shader 引用，`BufferUploader.drawWithShader` 取 `RenderSystem.getShader()` 并 `apply`。
- 着色器由 `GameRenderer` 在 reload 时从 `assets/minecraft/shaders/core/` 加载并注册到 `ShaderInstance` 缓存（`getRendertypeXxxShader` 系列方法）。

### 3.4 主渲染循环

- **入口**：`GameRenderer.render(partialTicks, nanoTime, renderLevel)`（`net/minecraft/client/renderer/GameRenderer.java`）→ `GameRenderer.renderLevel(...)` → 计算 camera/projection → `LevelRenderer.prepareCullFrustum` → **`LevelRenderer.renderLevel(...)`**。
- **`LevelRenderer.renderLevel`**（`net/minecraft/client/renderer/LevelRenderer.java`，核心调度）渲染顺序：
  1. **地形（不透明）**：
     - `renderChunkLayer(RenderType.solid(), ...)`
     - `renderChunkLayer(RenderType.cutoutMipped(), ...)`（树叶前临时关 mipmap）
     - `renderChunkLayer(RenderType.cutout(), ...)`
  2. **实体**：`profiler "entities"`，遍历 `level.entitiesForRendering()`，`shouldRender` 裁剪后 `renderEntity` → `entityRenderDispatcher.render`（写入 `MultiBufferSource.BufferSource`）。
     - 发光实体用 `OutlineBufferSource`。
     - 之后 `endLastBatch` + 按批 `endBatch(entitySolid/entityCutout/entityCutoutNoCull/entitySmoothCutout)`。
  3. **BlockEntity**：`profiler "blockentities"`，遍历 `CompiledChunk.getRenderableBlockEntities()`，平移到方块位置，处理破坏进度叠加层，调 `BlockEntityRendererDispatcher.render`。
  4. flush 一批 RenderType（`solid`/`endPortal`/`Sheets.solidBlockSheet`/`cutoutBlockSheet`/`bedSheet`/`shulkerBoxSheet`/`signSheet`/`chestSheet` 等）。
  5. **半透明**：`renderChunkLayer(RenderType.translucent(), ...)` + `renderChunkLayer(RenderType.tripwire(), ...)`。
  6. **粒子**：`particleEngine.render`。
  7. **天气/手持**等。
- **`LevelRenderer.renderChunkLayer(RenderType, poseStack, camX,camY,camZ, projection)`**（区块绘制）：
  - `renderType.setupRenderState()`（设置 shader/texture/blend 等）。
  - 若是 translucent：先做透明排序（`resortTransparency`，相机移动超过阈值时重算，每帧最多 15 个区块）。
  - 设置当前 shader 的 Uniform（MODEL_VIEW/PROJECTION/FOG/GAME_TIME/...）→ `shader.apply()`。
  - 取 `CHUNK_OFFSET` uniform。
  - 遍历 `renderChunksInFrustum`（translucent **反向**遍历保证从远到近）：
    - 若 `!compiledChunk.isEmpty(renderType)`：取 `VertexBuffer vb = chunk.getBuffer(renderType)`；
    - 设 `CHUNK_OFFSET = origin - camPos`（区块相对相机偏移）；
    - `vb.bind()` → `vb.draw()`（`glDrawElements`）。
  - `shader.clear()` + `VertexBuffer.unbind()` + `renderType.clearRenderState()`。
- **`LevelRenderer.renderEntity`** → `entityRenderDispatcher.render(entity, x-camX, y-camY, z-camZ, yaw, partialTick, poseStack, bufferSource, packedLight)`。

### 3.5 GPU 上传

方块（区块层）与实体走的上传路径不同：

- **区块层（静态 VBO）**：
  - 重建时 `ChunkRenderDispatcher.RenderChunk.RebuildTask.compile` 生成每层的 `RenderedBuffer` → `ChunkRenderDispatcher.uploadChunkLayer(renderedBuffer, vertexBuffer)` → `vertexBuffer.upload(renderedBuffer)`（在渲染线程异步执行）。
  - `upload` 把顶点数据 `glBufferData` 到 `vertexBufferId`（GL_ARRAY_BUFFER，STATIC），索引到 `indexBufferId`（或用共享的 sequential indices）。
  - 绘制：`LevelRenderer.renderChunkLayer` 中 `vb.bind()`（绑 VAO）+ `vb.draw()`（`glDrawElements`），通过 `CHUNK_OFFSET` uniform 实现区块定位（顶点数据存区块局部坐标）。
- **实体/即时（即时 VBO）**：
  - 数据先入 `MultiBufferSource.BufferSource` 的 `BufferBuilder`（CPU 堆外内存）。
  - `endBatch(renderType)` → `renderType.end(builder, sorting)` → `builder.end()` 成 `RenderedBuffer` → `setupRenderState()` → `BufferUploader.drawWithShader(renderedBuffer)`。
  - `BufferUploader.upload`：按 format 取 `VertexFormat.getImmediateDrawVertexBuffer()`（每种格式一个共享即时 VBO），`bind` + `upload`。
  - `drawWithShader`：`VertexBuffer._drawWithShader(modelView, projection, shader)`：设置 Uniform → `shader.apply()` → `draw()`（`glDrawElements`）→ `shader.clear()`。
- **索引**：QUADS 模式下若 `sequentialIndex`（无自定义排序），用 `RenderSystem.getSequentialBuffer(QUADS)` 提供的共享索引缓冲（自动扩容）；若有 `sortingPoints`（半透明排序）则上传自定义排序索引到 `indexBufferId`。

## 4. 数据流总结

### 4.1 方块管线（定义 → 屏幕）

```
JSON 模型文件 (assets/.../models/block/xxx.json)
   │  BlockModel.GSON (Deserializer: BlockModel/BlockElement/BlockElementFace/BlockFaceUV/ItemTransform/ItemTransforms/ItemOverride)
   ▼
BlockModel { elements:[BlockElement{from,to,faces:{Dir:BlockElementFace{cullForDirection,tintIndex,texture,uv:BlockFaceUV{uvs,rotation}}},rotation,shade}], textureMap, parent, transforms, overrides }
   │  ModelBakery (resolveParents → bake)  /  BlockModel.bakeVanilla
   │  遍历 element.faces → BlockModel.bakeFace → FaceBakery.bakeQuad
   │     FaceBakery.makeVertices → bakeVertex → fillVertex (写 int[32]: pos+color(-1)+uv0)
   │     → ForgeHooksClient.fillNormal (写 normal+padding)
   ▼
BakedModel (SimpleBakedModel: culledFaces[Dir], unculledFaces, particleIcon)  每个面 = BakedQuad{ int[32] vertices, tintIndex, direction, sprite, shade }
   │  BlockModelShaper.modelByStateCache  (BlockState → BakedModel)
   ▼
[区块重建]  ChunkRenderDispatcher.RenderChunk.RebuildTask.compile
   │  遍历 16³ 方块:
   │    BlockState → model.getRenderTypes (默认 ItemBlockRenderTypes.getChunkRenderType: solid/cutout/cutoutMipped/translucent/tripwire)
   │    FluidState → ItemBlockRenderTypes.getRenderLayer (translucent)
   │    → 对应 RenderType 的 BufferBuilder (begin: QUADS + BLOCK 格式)
   │    方块: BlockRenderDispatcher.renderBatched → ModelBlockRenderer.tesselateBlock
   │            → model.getQuads(dir) → putQuadData → VertexConsumer.putBulkData
   │            (读 BakedQuad.int[32], PoseStack 变换, 染色, 填 lightmap/overlay, 写 NEW_ENTITY 完整顶点...实际写 BLOCK 格式)
   │    流体: LiquidBlockRenderer.tesselate (直接 vertex(...))
   │    半透明: setQuadSorting (按距离排序)
   │  endOrDiscardIfEmpty → RenderedBuffer
   ▼
VertexBuffer.upload (glBufferData → GL_ARRAY_BUFFER/STATIC + 索引)  →  RenderChunk.buffers[RenderType]
   │  LevelRenderer.renderChunkLayer (solid→cutoutMipped→cutout ... translucent→tripwire)
   │    renderType.setupRenderState (shader+texture+blend); shader.apply; 设 CHUNK_OFFSET
   │    vb.bind (VAO); vb.draw (glDrawElements); renderType.clearRenderState
   ▼
屏幕像素
```

### 4.2 实体管线（定义 → 屏幕）

```
LayerDefinitions.createRoots()  (MeshDefinition + CubeListBuilder + PartPose 构建层级)
   │  EntityModelSet.bakeLayer → LayerDefinition.bakeRoot → PartDefinition.bake (递归)
   ▼
ModelPart 树 { cubes:[Cube{polygons:[Polygon{Vertex[4], normal}]}], children, x/y/z/xRot/yRot/zRot, visible }
   │  EntityRendererProvider.Context 取出 → 构造 EntityModel 子类 (如 HierarchicalModel, 默认 renderType=entityCutoutNoCull)
   ▼
[每帧]  EntityRenderDispatcher.render → LivingEntityRenderer.render
   │  PoseStack: setupRotations → scale(-1,-1,1) → scale(entity) → translate(0,-1.501,0)
   │  model.setupAnim (摆姿势: 改 ModelPart.xRot 等)
   │  getRenderType → buffer.getBuffer(renderType) → VertexConsumer
   │  model.renderToBuffer → ModelPart.render (递归: pushPose, translateAndRotate, compile)
   │    Cube.compile: 每顶点 vertexConsumer.vertex(x,y,z, r,g,b,a, u,v, overlay, lightmap, nx,ny,nz) (14 参数, NEW_ENTITY 格式)
   │  RenderLayer.render × N (盔甲/鞘翅/披风/眼睛/手持... 各取自己的 RenderType)
   ▼
MultiBufferSource.BufferSource (BufferBuilder, CPU 堆外内存, NEW_ENTITY 格式)
   │  endBatch(renderType): renderType.end(builder) → setupRenderState → BufferUploader.drawWithShader
   │    BufferUploader.upload (VertexFormat.getImmediateDrawVertexBuffer 共享即时 VBO) → drawWithShader
   │      VertexBuffer._drawWithShader: 设 Uniform(MODEL_VIEW/PROJECTION/...) → shader.apply → draw(glDrawElements)
   ▼
屏幕像素
```

### 4.3 关键差异对照

| 维度 | 方块（区块） | 实体 |
|------|------------|------|
| 模型来源 | JSON 文件 → Gson 反序列化 | 程序化 builder API（`MeshDefinition`） |
| 几何载体 | `BakedQuad`（int[32]，烘焙时预变换） | `ModelPart.Cube.Polygon`（运行时 PoseStack 变换） |
| 顶点格式 | `DefaultVertexFormat.BLOCK`（32B） | `DefaultVertexFormat.NEW_ENTITY`（36B，多 UV1 overlay） |
| VBO 类型 | 每 RenderType 一个静态 `VertexBuffer`（STATIC） | 即时 VBO（每 VertexFormat 一个，每帧重传） |
| RenderType 选择 | `BakedModel.getRenderTypes` / `ItemBlockRenderTypes`（chunk layer） | `model.renderType(texture)`（默认 `entityCutoutNoCull`）+ 各 RenderLayer 自选 |
| 绘制调度 | `LevelRenderer.renderChunkLayer`（遍历区块 VBO） | `MultiBufferSource.BufferSource.endBatch` |
| 变换时机 | 烘焙时做模型/元素变换；运行时仅 `CHUNK_OFFSET` uniform + AO | 运行时完整 `PoseStack`（旋转/缩放/平移）矩阵 |
| 批量写入 | `VertexConsumer.putBulkData`（读 BakedQuad） | `Cube.compile` 逐顶点 `vertex(...)` |
